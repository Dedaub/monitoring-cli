| Need | Snippet |
|------|---------|
| Selector of a call | `common.selector(td.calldata)` |
| Indexed address from `topic1`/`topic2` | `substring(log.topic1 FROM 13 FOR 20)::bytea` (right-most 20 of 32 bytes) |
| `uint256` from `log.data` | `common.hex_to_numeric('0x' || encode(log.data, 'hex'))` |
| `uint` packed in an indexed topic (e.g. UniV3 fee in `topic3`) | `get_byte(topic3::bytea, 29)*65536 + get_byte(topic3::bytea, 30)*256 + get_byte(topic3::bytea, 31)` ‚Äî cast `ethword`‚Üí`bytea`; last 3 bytes = `uint24` |
| **Readable address/hash in SELECT** | `concat('0x', encode(col, 'hex'))` (or `'0x' \|\| encode(col, 'hex')`) ‚Äî bare `encode()` has **no `0x`**; output-only, WHERE literals stay `\x` bytea. |
| **`tx_hash` in the result** | **Prefer the function `<chain>.tx_hash(block_number, tx_index)`** ‚Äî `concat('0x', encode(<chain>.tx_hash(x.block_number, x.tx_index),'hex'))`. **Verified present on all 7 chains** (ethereum/base/arbitrum/optimism/polygon/binance/avalanche), so use it on `logs`/`transaction_detail`/`token_ledger`/`token_transfers` instead of joining `outer_transaction`: one fewer table ‚Äî and it **keeps the large `outer_transaction` table out of the cold working set** (a real win on multi-chain UNIONs, where the cold set is the usual cause of a first-run timeout). `outer_transaction` still exposes `tx_hash` natively when it's already your driving table. Fallback (only if `validate-query` ever flags a missing `tx_hash()`): **JOIN** `{{outer_transaction(network=‚Ä¶, duration=‚Ä¶)}}` `ot ON ot.block_number = x.block_number AND ot.tx_index = x.tx_index` (1:1, **no fan-out**). |
| Token amount ‚Üí human units | divide by `pow(10, decimals)`. Source `decimals` from `latest_token_info` (row below); inline a constant only for a single pinned well-known token (USDC=6, WETH=18, WBTC=8, USDT=6). |
| **ERC20 metadata (`symbol`/`name`/`decimals`)** | JOIN `<chain>.latest_token_info lti ON lti.token_address = <token_addr_col>` ‚Äî `symbol`, `token_name`, `decimals` (smallint), `total_supply`. PK `token_address` ‚Üí **1:1, no fan-out**. Referenced raw (no macro). |
| **USD value (one-call convenience ‚Äî silently 0 for unpriced tokens)** | `<chain>.to_usd_value(raw_amount, token_address)` ‚Üí USD `numeric`, **decimals + price handled internally** (raw `1e18` WETH ‚Üí market $). One call, no manual `pow(10,decimals)`/price join. Per-chain function ‚Äî prefix the chain. Works on any numeric (incl. signed `value_delta`). **For an auditable USD figure prefer the explicit `network_token_info` LEFT JOIN (next row)** ‚Äî `last_price` is kept up to date there and a missing price surfaces as NULL instead of a silent 0; reserve `to_usd_value` for quick one-offs where every token is known-priced. **‚öÝ Returns `0` for any token with no `last_price` ‚Äî notably stablecoins** (USDC/USDT/DAI `last_price` is NULL; they are the USD quote unit): `to_usd_value(1e6, USDC)` ‚Üí **`0`**, NOT `1.00`. If a value leg can be a stablecoin, price it manually with the pinned-$1 COALESCE (next row), never this. |
| **USD price ‚Äî the proper, auditable join (+ decimals / symbol / logo)** | JOIN `<chain>.network_token_info nti ON nti.token_address = <token_addr_col>` (PK, 1:1): `last_price` (double, **kept up to date ‚Äî the canonical price source; join it for prices rather than leaning on `to_usd_value`'s hidden lookup**), `symbol`, `decimals`, `logo_small/medium/large`, `presentation_symbol`. (`latest_token_info` has **no price**.) Manual USD = `raw / pow(10, nti.decimals) * nti.last_price`; **cast for `round`:** `round((‚Ä¶ * nti.last_price)::numeric, 2)` (`last_price` is double; `round(double,int)` doesn't exist). **LEFT JOIN** so rows survive a missing price. **Stablecoin `last_price` is NULL** (USDC/USDT/DAI = the quote unit) ‚Üí pin to $1 **by address**: `COALESCE(nti.last_price::numeric, CASE WHEN <token_addr_col> = ANY(ARRAY['\x‚Ä¶usdc‚Ä¶'::bytea,'\x‚Ä¶usdt‚Ä¶'::bytea]) THEN 1.0 END)` ‚Äî **never** a blanket `COALESCE(‚Ä¶, 1)` (misvalues a genuinely unpriced *volatile* token at $1). Verify per chain which collaterals are NULL before trusting a USD sum. |
| **Live on-chain view call** | `{{ eth_call("<addr_col>.totalSupply()", outputs="( uint256 output0 )", network='<chain>') }}` reads current contract state at tip (‚Üí `<chain>.eth_call(addr, 'sig(returns)', jsonb_build_array(args))`). With args: `"<addr_col>.balanceOf(address owner)"`. **One RPC per row** ‚Äî use only on a **small** row set (post-aggregation, a param'd address, a `LIMIT`ed set), never across a big scan. For balances prefer `token_balance`. |
| **Holder balances / top-N / concentration** | `<chain>.token_balance` (raw, no macro), `value` = current raw balance. Holders of a token: `WHERE token_address='\x‚Ä¶' AND value > 0 ORDER BY value DESC LIMIT N` (token_address-leading PK). Balances of an address: `WHERE owner_address='\x‚Ä¶'` (owner index). **Add `value > 0`** for holder counts / concentration / registry-of-holders / supply sums ‚Äî `token_balance` is a never-pruned snapshot, so fully-exited holders persist as stale `value = 0` rows. Omit it only when a zero balance is the point (ex-holders, or investigating a specific token‚Üîwallet combo that holds nothing). No block-range needed ‚Äî current-state snapshot. |
| **Is an address a contract? / contract creation** | Classify: `JOIN {{contracts(network=‚Ä¶)}} c ON c.address = <addr_col>` (present ‚áí contract). A **contract-creation tx** has empty recipient: `outer_transaction.to_a = '\x'::bytea`. "Contract deployed a new contract" = creation tx whose `from_a` is in `contracts`. (Deployer‚Üídeployed enumeration is P11.) |
| **Contract code similarity (suspicious upgrade ‚Äî how much did the logic change?)** | Canonical = **decompiled-function embeddings**, not raw bytes: per implementation, `{{contracts(network=‚Ä¶)}} c ON c.address = <impl>` ‚Üí `c.md5_bytecode`, then `(SELECT avg(embedding) FROM common.decompiled_public_function_embeddings WHERE md5_bytecode = c.md5_bytecode)` per side, scored with `common.cosine_distance(old_emb, new_emb)` (‚âà0 = same logic, larger = rewrite). **NULL discipline:** a fresh implementation may not be decompiled/embedded yet ‚Üí a NULL embedding means **unknown, not similar** ‚Äî LEFT JOIN, surface the NULL, never COALESCE it away. Cheap pre-filters only: `c_old.md5_bytecode <> c_new.md5_bytecode` (changed at all); pg_trgm `similarity()` on bytecode hex / `decompiled_code.source_level` is a quick-and-dirty **fallback**, not the house method. Form old‚Üínew pairs with P2 on `Upgraded(address indexed implementation)` + `LAG(impl) OVER (PARTITION BY proxy ORDER BY block_number, log_index)`. |
| **chain_id literal ‚Äî function form** | `<chain>.get_chain_id()` returns the numeric chain id (`ethereum`‚Üí1, `base`‚Üí8453, ‚Ä¶) ‚Äî cleaner than hardcoding `8453 AS chain_id`. |
| **Mint / burn detection** | Transfers to/from the zero address: in `token_ledger`, `WHERE address = '\x0000000000000000000000000000000000000000'::bytea` ‚Üí `value_delta > 0` = **mint**, `value_delta < 0` = **burn**. Sum per period for net supply change. |
| **Deviation / drift between two prices (√∑0-safe)** | `abs(a - b) / GREATEST(abs(a), abs(b))`; alert when `> threshold` (e.g. `0.01` = 1%). `GREATEST` avoids divide-by-zero and normalizes both directions ‚Äî for oracle-vs-market or feed-vs-feed drift. |
| **1:1 enrich from a table that may match >1 row** | `LEFT JOIN LATERAL (SELECT col FROM {{token_ledger(...)}} tl WHERE tl.block_number=x.block_number AND tl.tx_index=x.tx_index AND tl.address='\x‚Ä¶' AND tl.value_delta = x.amount LIMIT 1) e ON true` ‚Äî the `LIMIT 1` inside guarantees **no fan-out** (safer than the match in a plain `JOIN ‚Ä¶ ON`). |
| **Force a topic0-led scan when a single pinned `address` would otherwise win** (rare event, busy contract, topic0-indexed chain) | A lone `address = '\x‚Ä¶'` (or a 1-element `IN`, which folds to `=`) makes the planner read *all* that contract's logs and over-estimate the row count (‚Üí `latest_token_info` seq-scans). Fence a **topic0-only** scan, filter the address outside: `FROM (SELECT block_number, tx_index, address, topic1, topic2, topic3, data FROM {{logs(network=‚Ä¶, duration=‚Ä¶)}} WHERE topic0 = '\x‚Ä¶' OFFSET 0) l ‚Ä¶ WHERE l.address = '\x‚Ä¶'`. **`OFFSET 0`** blocks predicate-pushdown/flattening so the inner scan uses the `(topic0,block_number,address)` index; the outer `address` filter still excludes fork emitters that share the topic0 (e.g. Aave-forks emitting Aave's `LiquidationCall`). Lighter than `‚Ä¶ AS MATERIALIZED` (no temp write). Only needed for a **single** pinned address ‚Äî for ‚â•2, `address = ANY(ARRAY[‚Ä¶])` already keeps topic0 leading. |
| **Token NOT carried in the event (resolve in-tx)** | When an event omits a token you need (e.g. Morpho `Liquidate` carries only market `id`) but the token *moves in the same tx*, read it from `token_ledger`: its row has `value_delta` magnitude = the event's amount. Use the LATERAL recipe above with `tl.address='\x<contract>'` and `tl.value_delta = <amount from event>` ‚Üí `tl.token_address` is the token (Morpho seized-collateral = `value_delta = +seizedAssets`; repaid loan = the `-repaidAssets` row). Resolves **in-window** ‚Äî no registry TABLE, no materialization. **Price each amount with ITS OWN token** (collateral vs debt vs Morpho `badDebtAssets`, which is *loan*-denominated). **Only works if the *underlying* moves:** cToken/eVault-share/aToken-share seizes (Compound V2 `seizeTokens`, Euler V2 `yieldBalance`, Aave V4 `liquidatedShares`) move a *wrapper* token that isn't USD-priced ‚Äî fall back to `eth_call` (`exchangeRateStored`/`convertToAssets`/reserveId‚Üíasset) to convert shares‚Üíunderlying. |
| **`chain_id` in the result** | Project a literal matching the network ‚Äî `8453` base, `1` ethereum, `42161` arbitrum, `10` optimism, `137` polygon, `56` bnb, `43114` avalanche, `81457` blast. In a cross-network `UNION`, each branch carries its own literal. |
| Time-bucket | `date_trunc('hour', <chain>.block_timestamp(block_number))` |

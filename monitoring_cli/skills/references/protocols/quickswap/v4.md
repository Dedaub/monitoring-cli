# QuickSwap V4 — Compressed Deployment Reference (Base / Algebra Integral)

**Status:** topic0/selectors computed locally via keccak-256; **core pool topics + init code hash confirmed against live `eth_getLogs`/factory `POOL_INIT_CODE_HASH()`** on the Base WETH/USDC pool (2026-06-05, publicnode); addresses verified via `eth_getCode`/`eth_call`. Pool init hash verified by CREATE2 round-trip of the live pool.
**Scope:** QuickSwap **"V4"** — built on **Algebra Integral** (the plugin-based, post-V1 engine; NFT name "Algebra Positions NFT-**V2**"). Distinct from [`v3.md`](v3.md) (Algebra V1, Polygon) and [`v2.md`](v2.md) (Uniswap V2 fork).
**Key facts:**
- Among the 7 requested chains, QuickSwap V4 is **deployed only on Base (8453)**. (QuickSwap's other V4 Algebra-Integral chains — Soneium, Somnia, Mantra — are out of this set. Base also keeps a separate classic V2 AMM, see [`v2.md`](v2.md).)
- **Integral is plugin-based**: tick-spacing, fee, and the plugin set are mutable per-pool. The dynamic fee/TWAP oracle lives in an external **plugin** (not in the pool as in V1). `globalState()` changed shape (dropped `timepointIndex`, added `pluginConfig`).
- **This Base deployment is a NEWER Integral than the generic `integral-v1.2.2`:** the live **Swap carries two extra `uint24` fees** (`overrideFee`, `pluginFee`) and **Burn carries a trailing `uint24`** — both verified on-chain. Its **init code hash is QuickSwap-specific** (`0xa18736c3…`, NOT the generic Integral `0x62441ebe…`) — read `POOL_INIT_CODE_HASH()` per deployment, never assume.

## Proxies

**Core immutable** (factory, pool deployer, pools, router, NFPM are non-upgradeable; pools CREATE2 with init hash `0xa18736c3…`, `abi.encode` salt). **The plugin layer is the mutable surface:** each pool has a `plugin()` (dynamic-fee + volatility-oracle contract) installed by a plugin factory; `owner()` (`0xa08dac76…`, shared QuickSwap governance) can set defaults, swap the `DefaultPluginFactory`, change tick-spacing/fee/community-fee, and the `farmingAddress`. Pools also support `createCustomPool` with a different salt scheme (`keccak256(abi.encode(deployer, token0, token1))`). The volatility-oracle TWAP writes timepoints **silently — no oracle event** is emitted.

## Detection invariants & gotchas

1. **Base-V4 Swap topic `0x121cb44e…` is unique** (carries `overrideFee`+`pluginFee`) → a single Swap topic self-identifies a QuickSwap-V4-on-Base pool, with no collision against Uniswap V3 / Algebra V1. Effective fee for the swap = the `overrideFee`/`pluginFee` fields (1e-6 units), NOT a fixed tier.
2. **Burn here carries a trailing `uint24`** (`0x932214d4…`) but **Mint does NOT** (classic 7-arg `0x7a53080b…`) — an asymmetry specific to this Integral build; verified live. Decode accordingly.
3. **`globalState()` is the Integral tuple** (`price, tick, lastFee, pluginConfig, communityFee, unlocked`) — do NOT decode it with the V1 7-field layout even though the selector `0xe76c01e4` is identical.
4. **Init code hash is deployment-specific.** Read `POOL_INIT_CODE_HASH()` (`0xdc6fd8ab`) from the factory; this Base build's `0xa18736c3…` ≠ generic Integral `0x62441ebe…`. Use **PoolDeployer** (not factory) as the CREATE2 deployer.
5. **Plugin/oracle emits no events** — TWAP/volatility timepoints are silent storage writes. There is no oracle topic to monitor; read via the plugin contract if needed.
6. **NFT is "Algebra Positions NFT-V2"** (`ALGB-POS`) — the "V2" in the name marks Integral, distinguishing it from Polygon V3's "NFT-V1"; the **symbol is identical** (`ALGB-POS`) on both, so disambiguate by chain/factory, not symbol.
7. **Config events (TickSpacing/Plugin/PluginConfig/Fee/CommunityFee/CustomPool/Default*) are source-derived**, not live-confirmed on the sampled pool (they fire only on admin/config changes). Recompute/confirm against an actual emission before keying alerts on them.
8. **Base has BOTH** a classic V2 AMM (`v2.md`) and this V4 Algebra — different factories, ABIs, and init hashes. Don't conflate.

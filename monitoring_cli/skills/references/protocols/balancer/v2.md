# Balancer V2 — Topics, Selectors, Addresses (Ethereum, Arbitrum, Optimism, Base, Polygon, Gnosis, Avalanche)

**Status:** event topic0 hashes and function selectors computed locally with `cast keccak` / `cast sig` from canonical signatures read out of `balancer/balancer-v2-monorepo` (verified — V2 Vault `Swap` and the four Vault selectors match known anchors). Addresses from the official `balancer/balancer-deployments` registry, then **independently re-verified on-chain** via `cast` against `publicnode` RPCs on all seven chains (2026-05). See §10.
**Scope:** Balancer **V2** — the Vault, pools, factories, and the (version-agnostic) BAL/veBAL/gauge tokenomics stack. Balancer **V3** is a separate deployment documented in the sibling [`v3.md`](v3.md). Topics + selectors are chain-agnostic; addresses are network-specific.

**The one fact that defines Balancer monitoring: there is a single `Vault` contract, and (almost) every swap, join, exit, and flash loan emits its event from the Vault — not from the individual pool.** A Balancer pool holds no tokens itself; the Vault (`0xBA12…F2C8`, same on every chain) custodies all assets for all pools. So you monitor Balancer V2 by watching **one address** and filtering by `topic0` + `poolId`, rather than discovering and watching thousands of pool contracts. Pools are still individual contracts (the BPT ERC-20, deployed by factories) and emit a few of their own events (fee/amp/pause changes), but the trade/liquidity flow is all Vault-side.

> The `Swap` event also exists in V3 but hashes to a **different** `topic0` and carries different fields — never share a constant between versions. See [`v3.md`](v3.md).

## 11. Proxies

- **V2 Vault is a monolithic singleton — not a proxy and not upgradeable.** Its logic is fixed; only the `Authorizer` (permission module) can be swapped, surfaced by `AuthorizerChanged`. That swaps *who can call privileged functions*, not the Vault's swap/accounting logic.
- **V2 pools are full contracts deployed by factories** (not EIP-1167 clones). Logic is immutable; pools support pause and recovery mode (`PausedStateChanged`, `RecoveryModeStateChanged`) but no code upgrade. Enumerate pools via `PoolCreated` / the Vault's `PoolRegistered` — there is no CREATE2 init-code-hash shortcut like Uniswap V2.

See `references/proxies.md` for general proxy detection, and [`v3.md`](v3.md) §Proxies for the V3 Vault's delegatecall split.

## 12. Detection invariants & gotchas

1. **Watch the Vault, not the pools.** `Swap`/`PoolBalanceChanged`/`FlashLoan` all emit from `0xBA12…F2C8`. Filtering by `(vault address, topic0)` captures all pools at once — you do not need to discover pool addresses to see their trades.
2. **`poolId` embeds the pool address.** The leftmost 20 bytes of the 32-byte `poolId` are the pool contract address: `pool = substring(poolId from 1 for 20)`. Bytes 21–22 are the `specialization`, the rest a nonce. No `PoolRegistered` lookup needed to map a `Swap` to its pool.
3. **`PoolBalanceChanged` is join AND exit.** Sign of the `deltas[]` distinguishes: positive = join (tokens in), negative = exit. A naive "join" detector keying on the event name alone double-counts exits.
4. **`batchSwap` emits one `Swap` per hop**, all in the same tx from the Vault. A single user trade across N pools = N `Swap` logs. Aggregate by tx + the `FundManagement.sender`/`tx.origin`, not per-log.
5. **`SwapFeePercentageChanged` is the pool's `(uint256)` form** here (topic0 `0xa9ba3ffe…`), emitted by the pool. The V3 `(address,uint256)` form is emitted by the Vault — different topic0 (`v3.md`).
6. **ComposableStable "phantom BPT" swaps.** In composable stable pools the BPT is itself a registered token, so joins/exits can be executed as a `Swap` involving the BPT. A "swap volume" metric that includes these double-counts liquidity events as trades — exclude swaps where one side is the pool's own BPT.
7. **veBAL `Deposit` (`0x4566dfc2…`, 5 fields) vs gauge `Deposit` (`0xe1fffcc4…`, 2 fields)** are different events with the same name — these topic0s are also shared with Curve's veCRV/gauge (Balancer forked them). Always disambiguate by contract address.
8. **BAL token address gotcha.** The canonical BAL is `0xba100000625a3754423978a60c9317c58a424e3D`. The `balancer-deployments` registry also lists a `TestBalancerToken` (`0x397BC4…`) whose `symbol()` is *also* `"BAL"` — do not use it. (Confirmed on-chain: both return `"BAL"`; the canonical one is correct.)
9. **The swap fee is encoded, not a transfer.** V2 takes the fee inside the Vault; protocol fees accrue to the `ProtocolFeesCollector`.
10. **Factories don't disappear.** Old pool-factory versions remain deployed and can still hold pools; `FactoryDisabled` only blocks new creations. To enumerate all pools of a type, union `PoolCreated` across every factory version on the chain.
11. **Bytea hex literals must have an even digit count:** 40 hex chars for addresses, 64 for 32-byte topic/`poolId` values.

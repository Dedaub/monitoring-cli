# Fraxswap V1 — Deprecated (Topics, Selectors, Addresses)

**Status:** DEPRECATED. V1 factories remain deployed on-chain (bytecode confirmed; contracts do not self-destruct) but Frax Finance considers V1 superseded by V2. No new pools should be created via V1 factories; existing V1 pools are legacy only. Addresses sourced from Frax Finance docs (https://docs.frax.finance/fraxswap/fraxswap-contract-addresses); factory existence confirmed via `eth_getCode` on Ethereum (`0xB076…Be4b`: 48,290 chars), Arbitrum (`0x5Ca1…e08E`: 48,290 chars), and Optimism (`0xBe90…0437`: 48,290 chars) on 2026-06-09.
**Scope:** Fraxswap V1 — the original TWAMM AMM built on a Uniswap V2 base, deployed on Ethereum, BNB Smart Chain, Avalanche, Arbitrum, Optimism, and Polygon (NOT Base — V1 predates Base). V1 has a **fixed 1% fee (100 bps) on every pool**; this is the primary functional difference from V2, which allows per-pool configurable fees. V2 otherwise shares the same TWAMM core and the same Uniswap V2 base — topics and selectors are identical between versions.

---

## 1. Topics

V1 topics are **identical to V2** — same Uniswap V2 base events (`Swap`, `Sync`, `Mint`, `Burn`, `Transfer`, `Approval`) and same TWAMM-specific events (`LongTermSwapSubmitted`, `CancelLongTermOrder`, `LongTermSwapPerformed`, `OrderExpired`, `LiquidityFeeUpdated`, etc.). Do not re-enumerate here — see [v2.md](v2.md) §1 for all topic0 values. When filtering historical logs from V1 factory-deployed pairs, the same topic0 set applies; distinguish V1 vs. V2 pairs by factory origin (see §3).

---

## 2. Function signatures

V1 selectors are **identical to V2** — same Uniswap V2 pair interface plus same TWAMM order functions (`longTermSwapTokenToToken`, `cancelLongTermSwap`, `executeVirtualOrders`, etc.). See [v2.md](v2.md) §2 for all selectors. The sole behavioral difference at the ABI level: V1 pairs were deployed with a hardcoded 1% fee; V2 pairs carry a configurable `fee` storage variable.

---

## 4. Proxies

V1 factories and pairs are **not upgradeable proxies** — they are immutable deployments (same as Uniswap V2). No proxy pattern; no EIP-1967 impl slot. The factory is a plain contract; each pair is deployed as a minimal clone or direct deploy. There is nothing to track for upgrades on V1.

---

## 5. Verification & sources

- **Factory existence:** `eth_getCode` confirmed non-empty bytecode on Ethereum `0xB076…Be4b`, Arbitrum `0x5Ca1…e08E`, and Optimism `0xBe90…0437` via `https://ethereum-rpc.publicnode.com`, `https://arbitrum-one-rpc.publicnode.com`, `https://optimism-rpc.publicnode.com` on 2026-06-09.
- **Factory addresses:** Frax Finance official docs — https://docs.frax.finance/fraxswap/fraxswap-contract-addresses (V1 section, entries marked as outdated/deprecated).
- **Topics & selectors:** identical to V2; see [v2.md](v2.md) for all verified values.
- **Key V1 vs. V2 difference:** fixed 1% fee (100 bps) in V1 vs. per-pool configurable fee in V2 — confirmed by Frax docs and Fraxswap GitHub (`FraxswapFactory` V2 adds `fee` parameter to `createPair`).
- **Base absence:** V1 was launched before Base existed; Frax docs list no V1 factory for Base. V2 is the only Fraxswap version on Base.

# Maple V1 ("Maple 1.0", `maple-core`) — Topics, Selectors, Addresses (Ethereum only)

**Status:** verified 2026-05-29 against the canonical [`maple-labs/maple-core`](https://github.com/maple-labs/maple-core) source (cloned `main`), live `eth_getCode` / `eth_call` / `eth_getLogs` on `https://ethereum-rpc.publicnode.com`, and local keccak. Every topic0 below was recomputed locally and cross-checked against deployed bytecode (event topic constants appear as `PUSH32` immediates in the emitting contract / library).
**Scope:** Maple V1 was **only ever deployed on Ethereum mainnet (chain ID 1).** No Base/BNB/Arbitrum/Optimism/Polygon/Avalanche V1. It is **fully deprecated** — superseded by the Dec-2022 "Maple 2.0" rewrite ([v2.md](v2.md)). All V1 pools are wound down; expect near-zero recent activity. Keep V1 and 2.0 strictly separate: different repos (`maple-core` vs `maple-core-v2`), different contracts, different events.

> **MPL and xMPL are shared across V1 and 2.0.** `MPL` (`0x3334…935e6`) was the V1 governance/fee token; it has since been migrated 1:100 to **SYRUP** (see [syrup-cross-chain.md](syrup-cross-chain.md)). `xMPL` (`0x4937…2914c`) is the MPL staking wrapper. Do not confuse V1 `MapleGlobals` (`0xC234…3600`, non-proxied) with the 2.0 Globals proxy (`0x804a…390C`).

## 1. Architecture

V1 is a **pool-delegate** model: each delegate runs a `Pool` that lenders deposit into; the Pool funds under-collateralized `Loan`s to whitelisted borrowers. First-loss capital is **Balancer BPT** (MPL/USDC 50/50) locked in a `StakeLocker`. Per-(pool,loan) accounting is custodied in a `DebtLocker`. Funds/collateral are held in dedicated lockers. **Math is FDT (Funds Distribution Token, ERC-2222)** — both Pool LP tokens and Loan tokens are FDTs; lenders "claim" pro-rata interest. Core logic is delegated to the `PoolLib` / `LoanLib` / `Util` libraries (events fired there land on the calling Pool/Loan address).

| Contract | Role |
|----------|------|
| **MapleGlobals** | Non-proxied registry/access hub: governor, valid pool delegates, valid assets/oracles, fees, pause. |
| **PoolFactory** | Deploys `Pool`s (validated delegates only). Enumerable via `poolsCreated()`/`pools(i)`. |
| **Pool** | One delegate-run lending pool; holds LP deposits, funds Loans, distributes interest via `claim`. ERC-2222 LP token ("MPL-LP"). |
| **LoanFactory** | Deploys `Loan`s. Enumerable via `loansCreated()`/`loans(i)`. |
| **Loan** | One borrower loan: drawdown, scheduled payments, default/liquidation. ERC-2222. |
| **DebtLocker / DebtLockerFactory** | Per-(pool,loan) custody of the Pool's loan position (claims repayments, triggers default). |
| **StakeLocker / StakeLockerFactory** | Custodies delegate+staker BPT first-loss capital; BPT burned on default. |
| **LiquidityLocker / CollateralLocker / FundingLocker (+ factories)** | Hold pool liquidity / loan collateral / pre-drawdown funds. |
| **MapleTreasury** | Receives protocol fees, routes to MPL holders. |
| **MplRewards / MplRewardsFactory** | Synthetix-style MPL emissions to MPL-LP / BPT stakers. |
| **RepaymentCalc / LateFeeCalc / PremiumCalc** | Pluggable interest/fee/premium calculators referenced by each Loan. |
| **MapleToken (MPL)** | ERC-2222 governance + fee-share token (migrated → SYRUP). |
| **xMPL** | MPL staking wrapper (ERC-4626-like, accrues treasury revenue). |
| **BPool** | Balancer V1 MPL/USDC 50/50 pool; its BPT is the staking/first-loss asset. |

## 5. Proxies

**V1 is almost entirely non-proxied** (immutable contracts deployed per instance by the factories). The lockers and the Pool/Loan instances are plain deployments, not upgradeable proxies — there is **no EIP-1967 impl slot** to read (verified: reading the 1967 slot on V1 contracts returns zero). Upgrades in V1 happened by deploying new factory/calc versions and pointing `MapleGlobals` at them, not by proxy upgrade. `xMPL` is the one exception with richer logic but is also non-proxied. Contrast with 2.0, which is fully proxied ([v2.md §7](v2.md)).

## 6. Detection invariants & gotchas

1. **Ethereum-only, deprecated.** No V1 on any L2/sidechain. Activity is ~zero now.
2. **`LoanFunded` is overloaded across Pool and Loan with different signatures → different topic0** (`0xf62badb0…` Pool vs `0x726d5f1a…` Loan). Match on `(topic0, emitter)`.
3. **`BalanceUpdated` shares one topic0** (`0x2047d163…`) across Pool and Loan (identical types) — disambiguate by emitter.
4. **No `Deposit`/`Withdraw` events** — infer lender flow from MPL-LP `Transfer` (mint/burn) + `BalanceUpdated`.
5. **Library-emitted events:** Pool `LoanFunded` is emitted inside `PoolLib` via `delegatecall`, so the topic constant lives in PoolLib bytecode but the log's `address` is the **Pool** (delegatecall preserves context).
6. **State enums hash as `uint8`:** `PoolStateChanged(uint8)`, `LoanStateChanged(uint8)`.
7. **MPL/xMPL/Treasury are shared with 2.0** — seeing MPL `Transfer`s does not imply V1 activity; MPL is being migrated to SYRUP.
8. **First-loss is Balancer BPT, not the pool asset** — `DefaultSuffered` reports `bptsBurned`.

# Compound V2 — Topics, Selectors, Addresses (Ethereum only)

**Status:** verified against Ethereum mainnet RPC (`ethereum-rpc.publicnode.com`) and the `compound-finance/compound-protocol` repo on 2026-05-29. Every topic0/selector computed locally with keccak and cross-checked against live `eth_getLogs` (cUSDC, cUSDC) and deployed bytecode.
**Scope:** Compound V2 is the cToken + Comptroller lending protocol. **Canonical Compound-Labs V2 is deployed on Ethereum mainnet ONLY.** It was never deployed by Compound Labs to Base, BNB, Avalanche, Arbitrum, Optimism, or Polygon — the well-known "Compound forks" on those chains (Venus on BNB, Benqi on Avalanche, Sonne/Moonwell on OP-Stack, etc.) are *separate protocols* with different addresses, different oracle wiring, and sometimes different event signatures. **Do not treat a fork as Compound.** The cross-chain Compound story is entirely **V3 / Comet** — see `compound/v3.md`.

Compound V2 is an upgradeable system built on two custom proxy patterns (NOT EIP-1967): the **Unitroller** (Comptroller proxy) and per-market **CErc20Delegator** (cToken proxy). Not every market is a delegator: the original markets that predate the Delegator pattern are immutable `CErc20` contracts (cUSDC, legacy cWBTC, cBAT, cZRX, cREP, cSAI — `implementation()` reverts on each), and `cETH` is an immutable `CEther`. Newer/re-listed markets (cDAI, cUSDT, cWBTC2, cUNI, cCOMP, cLINK, cAAVE, cSUSHI, cTUSD, cMKR, cYFI, cUSDP, cFEI, …) are `CErc20Delegator` proxies that delegate to a shared `CErc20Delegate` logic. Interest accrues **per block** (`borrowRatePerBlock`), the legacy design that V3 replaced with per-second accrual. The protocol still uses the original **error-code return convention**: many state-changing functions return a `uint` error code (0 = success) and emit a `Failure(error, info, detail)` event instead of reverting — a critical monitoring gotcha (a "successful" tx can be a silent no-op).

## 4. Proxies

Compound V2 predates EIP-1967 and uses **two bespoke delegatecall-proxy patterns**. The standard EIP-1967 impl slot (`0x3608…bbc`) is **empty** for every V2 contract — do not probe it; read the typed getters instead.

| Contract | Proxy pattern | Read current impl | Upgrade path |
|----------|---------------|-------------------|--------------|
| **Comptroller** | `Unitroller` — storage contract; `fallback` delegatecalls `comptrollerImplementation`. Impl stored in a plain storage var (not EIP-1967). | `comptrollerImplementation()` → `0xbb82aa5e` | `_setPendingImplementation` → impl calls `_become(unitroller)`. Two-step. |
| **cERC20 markets (delegators)** | `CErc20Delegator` — `fallback` delegatecalls `implementation`. Plain storage var. Covers the newer/re-listed markets (cDAI, cUSDT, cWBTC2, cUNI, cCOMP, cLINK, cAAVE, cSUSHI, cTUSD, cMKR, cYFI, cUSDP, cFEI, …). | `implementation()` → `0x5c60da1b` | `_setImplementation(impl, allowResign, becomeData)`. Emits `NewImplementation`. |
| **Original cERC20 markets (immutable)** | **None** — plain immutable `CErc20` predating the Delegator pattern: cUSDC, legacy cWBTC, cBAT, cZRX, cREP, cSAI. `implementation()` reverts. | n/a (revert) | Not upgradeable. |
| **cETH** | **None** — `CEther` is a plain immutable contract. | n/a | Not upgradeable. |
| GovernorBravo | Custom `GovernorBravoDelegator` (same delegatecall idiom). | `implementation()` | Governance. |

**Detection:** to enumerate the live logic for a delegator market, call `implementation()` on the cToken. This reverts on the immutable markets (cUSDC, legacy cWBTC, cBAT, cZRX, cREP, cSAI) and on cETH — a revert means the market is an immutable `CErc20`/`CEther` and *is* its own logic, not that detection failed. To get the live Comptroller logic, call `comptrollerImplementation()` on the Unitroller. Neither is at the EIP-1967 slot.

## 5. Detection invariants & gotchas

1. **Ethereum-only.** Canonical Compound V2 has **no deployment** on Base/BNB/Avalanche/Arbitrum/Optimism/Polygon. Any cToken-shaped contract there is a fork (Venus, Benqi, Sonne, Moonwell, Tarot…) — different addresses/oracles, sometimes mutated events. Don't index forks as Compound.
2. **`Failure` is a silent no-op signal.** A `mint`/`borrow`/`redeem`/`liquidate` tx that emits `Failure(error, info, detail)` (`0x45b9…9aa0`) did NOT move funds — the legacy error-code convention returns instead of reverting. Treat presence of `Failure` in a market's logs as a failed action.
3. **Two `AccrueInterest` signatures.** 3-arg (`0x8753…6cb9`, original cTokens incl. cUSDC) and 4-arg (`0x4dec…bc04`, newer delegates). Index both.
4. **cToken decimals = 8 always** (not the underlying's). Underlying value = `cTokenAmount × exchangeRateStored`, and `exchangeRateStored` itself is scaled by `1e(18 - 8 + underlyingDecimals)`.
5. **Per-block rates, not per-second.** `supplyRatePerBlock`/`borrowRatePerBlock`. Annualize with blocks/year (~2,628,000 at 12s). V3 switched to per-second — don't mix the two.
6. **cETH is special.** No `underlying()`; uses payable `mint()`/`repayBorrow()`; immutable (not a delegator). Detection code that always calls `underlying()` will revert on cETH.
7. **Liquidation emits `LiquidateBorrow` on the debt cToken + `Transfer`/`seize` on the collateral cToken** in the same tx. The collateral cToken's `seize` (`0xb2a02ff1`) moves shares to the liquidator at `liquidationIncentiveMantissa`.
8. **COMP rewards are pull-based** via `claimComp`; accrual is tracked off the `DistributedSupplierComp`/`DistributedBorrowerComp` events. COMP isn't auto-sent on supply/borrow.
9. **Borrow indexing is account-relative.** `borrowBalanceStored` already includes accrued interest via `borrowIndex`; don't double-count from `Borrow.borrowAmount`.
10. **Same COMP token feeds V3.** `0xc00e…6888` is shared between V2 Comptroller distributions and V3 `CometRewards` on Ethereum.

## 7. Verification & sources

- **Topics + selectors:** computed locally as `keccak256(canonical signature)` / `[0:4]` (pycryptodome). Cross-checked against live `eth_getLogs` on cUSDC (`0x39AA…7563`) — observed `Transfer`, `AccrueInterest`(3-arg), `Redeem`, `RepayBorrow`, `LiquidateBorrow`, `Failure` — and on cUSDCv3-adjacent contracts. cToken selectors confirmed present in the live `CErc20Delegate` bytecode (`0x3363bae2…`).
- **Addresses:** every cToken + core contract returns non-empty `eth_getCode` on Ethereum mainnet (2026-05-29). Comptroller logic read live via `comptrollerImplementation()` = `0xbafe01ff…b526`; cDAI delegate via `implementation()` = `0x3363bae2…a376`.
- **Absence on other chains:** `eth_getCode(Unitroller)` and `eth_getCode(cUSDCv3)` both empty on BNB (56) and Avalanche (43114). Compound Labs never deployed V2 off Ethereum.
- Authoritative source: [`compound-finance/compound-protocol`](https://github.com/compound-finance/compound-protocol) (CToken.sol, Comptroller.sol, Unitroller.sol, CErc20Delegator.sol, Governance/).

# Uniswap V2 — Topics, Selectors, Addresses (Ethereum, Base, BNB, Avalanche, Arbitrum, Optimism, Polygon)

**Status:** verified against Ethereum, Base, BNB, Avalanche, Arbitrum, Optimism, and Polygon mainnet RPC and the canonical `Uniswap/v2-core` + `Uniswap/v2-periphery` repos on 2026-05-21.
**Scope:** all seven chains where Uniswap Labs deployed V2. Topics + selectors are chain-agnostic; addresses are network-specific.

Uniswap V2 is **fully immutable** — Factory, Routers (01 and 02), Migrator, and every Pair are deployed without proxies and cannot be upgraded. Pairs are deployed by `UniswapV2Factory.createPair(tokenA, tokenB)` via `CREATE2` with `salt = keccak256(abi.encodePacked(token0, token1))` and `token0 < token1` enforced. **The pool init code hash is identical on every Uniswap-Labs V2 deployment:** `0x96e8ac4277198ff8b6f785478aa9a39f403cb768dd02cbee326c3e7da348845f` — verified by CREATE2 round-trip of the live `WrappedNative/USDC` pair on each of the seven chains. Each pair is BOTH an AMM exchange AND an ERC-20 LP token (`name = "Uniswap V2"`, `symbol = "UNI-V2"`, 18 decimals, with EIP-2612 permit). The 30 bps swap fee is taken inside `swap()` (no separate event); the optional 1/6-of-fee protocol cut is minted as additional LP to `feeTo` inside `mint()` / `burn()` when `feeTo != address(0)`. Protocol fees are **ON** on Ethereum / Base / Arbitrum / Optimism / BNB / Polygon and **OFF** only on Avalanche as of 2026-06.

## 1. Topics (chain-agnostic — `topic0 = keccak256(event signature)`)

### 1.1 UniswapV2Factory

| topic0 | Event |
|--------|-------|
| `0x0d3648bd0f6ba80134a33ba9275ac585d9d315f0ad8355cddefde31afa28d0e9` | `PairCreated(address indexed token0, address indexed token1, address pair, uint256)` |

Emitted once per `createPair()`. The trailing `uint256` is the new `allPairs.length` (1-indexed). `pair` and the count live in `data`. **`token0 < token1`** is enforced — the unsorted order is normalized before the event.

### 1.2 UniswapV2Pair (AMM events)

| topic0 | Event |
|--------|-------|
| `0x4c209b5fc8ad50758f13e2e1088ba56a560dff690a1c6fef26394f4c03821c4f` | `Mint(address indexed sender, uint256 amount0, uint256 amount1)` |
| `0xdccd412f0b1252819cb1fd330b93224ca42612892bb3f4f789976e6d81936496` | `Burn(address indexed sender, uint256 amount0, uint256 amount1, address indexed to)` |
| `0xd78ad95fa46c994b6551d0da85fc275fe613ce37657fb8d5e3d130840159d822` | `Swap(address indexed sender, uint256 amount0In, uint256 amount1In, uint256 amount0Out, uint256 amount1Out, address indexed to)` |
| `0x1c411e9a96e071241c2f21f7726b17ae89e3cab4c78be50e062b03a9fffbbad1` | `Sync(uint112 reserve0, uint112 reserve1)` |

`Swap` is the workhorse: filter on `topic0` + pair address. Exactly one `Mint`/`Burn`/`Swap`/`Sync`/`Skim`-call → one `Sync`. `Mint`/`Burn` `sender` is the Router (or whoever called `pair.mint/burn`).

### 1.3 UniswapV2Pair (ERC-20 LP token events)

| topic0 | Event |
|--------|-------|
| `0xddf252ad1be2c89b69c2b068fc378daa952ba7f163c4a11628f55a4df523b3ef` | `Transfer(address indexed from, address indexed to, uint256 value)` |
| `0x8c5be1e5ebec7d5bd14f71427d1e84f3dd0314c0f7b2291e5b200ac8c7c3b925` | `Approval(address indexed owner, address indexed spender, uint256 value)` |

`Transfer(0x0 → 0x0, 1000)` on the first `mint` of a pair is the permanent `MINIMUM_LIQUIDITY` lock. `Approval` is also emitted by EIP-2612 `permit()`.

### 1.4 EIP-712 typehashes (Pair LP token permit)

| Value | Constant |
|--------|-----------|
| `0x6e71edae12b1b97f4d1f60370fef10105fa2faae0126114a169c64845d6126c9` | `PERMIT_TYPEHASH = keccak256("Permit(address owner,address spender,uint256 value,uint256 nonce,uint256 deadline)")` |
| `0x8b73c3c69bb8fe3d512ecc4cf759cc79239f7b179b0ffacaa9a75d522b39400f` | `EIP712Domain` typehash — domain `name="Uniswap V2"`, `version="1"`, `verifyingContract = pair`. `DOMAIN_SEPARATOR` is per-pair (read via `0x3644e515`). |

## 2. Function signatures (chain-agnostic)

Selectors below are `keccak256(canonical signature)[0:4]`. All verified against deployed dispatcher bytecode and the published source in `Uniswap/v2-core` / `Uniswap/v2-periphery`.

### 2.1 UniswapV2Factory

| Selector | Signature | Returns / mutates |
|----------|-----------|--------------------|
| `0xc9c65396` | `createPair(address tokenA, address tokenB)` | Returns `address pair`. Emits `PairCreated`. CREATE2-deploys the pair contract. |
| `0xe6a43905` | `getPair(address, address)` | Public mapping — symmetric (`getPair[a][b] == getPair[b][a]`). Returns `address(0)` if no pair. |
| `0x1e3dd18b` | `allPairs(uint256)` | Public array of every pair ever created. |
| `0x574f2ba3` | `allPairsLength()` | `uint256` — count of all pairs on this chain. |

| `0xf46901ed` | `setFeeTo(address)` | Only `feeToSetter`. |
| `0xa2e74af6` | `setFeeToSetter(address)` | Only `feeToSetter`. Rotates control. |

## 11. Proxies

**There are no proxies in Uniswap V2 on any chain.** The Factory, Routers (01 and 02), Migrator, and every Pair are immutable. The Pair is `CREATE2`-deployed by the Factory but is not a minimal-proxy / clone — it embeds the full Pair bytecode each time.

| Item | Pattern | Why this matters |
|------|---------|------------------|
| `UniswapV2Pair` creation | `CREATE2(factory, salt = keccak256(token0,token1), init = type(UniswapV2Pair).creationCode)` | Pair address is fully deterministic from `(factory, token0, token1)` + init code hash → derive off-chain; no need to index `PairCreated`. |
| Pair admin / upgrade | **None** | Once deployed, a pair is frozen forever. Fee rate (30 bps), storage layout, `MINIMUM_LIQUIDITY` immutable. |
| `feeTo` / `feeToSetter` | Plain `address` state | The only mutable knob in the entire V2 stack — and it only redirects the protocol-fee LP-mint destination. Cannot change AMM logic. |

Pool init code hash is **identical on all 7 supported chains** (verified by CREATE2 round-trip on each):

```
UniswapV2Pair init code hash (Ethereum, Base, BNB, Avalanche, Arbitrum, Optimism, Polygon):
0x96e8ac4277198ff8b6f785478aa9a39f403cb768dd02cbee326c3e7da348845f
```

Off-chain derivation (works on every chain by swapping `factory_addr`):

```sql
WITH sorted AS (
  SELECT LEAST(tokenA, tokenB)::bytea AS token0,
         GREATEST(tokenA, tokenB)::bytea AS token1
)
SELECT substring(
  common.keccak256(
    '\xff'::bytea || factory_addr
    || common.keccak256(token0 || token1)
    || '\x96e8ac4277198ff8b6f785478aa9a39f403cb768dd02cbee326c3e7da348845f'::bytea
  ) FROM 13 FOR 20
) AS pair
FROM sorted;
```

## 12. Detection invariants & gotchas

1. **Swap = `Swap` + `Sync` from the same pair in the same tx.** A `Swap` without `Sync` is malformed. Use `Sync` to recover post-trade reserves at log granularity.
2. **`Mint` / `Burn` are always preceded by ERC-20 `Transfer`s** into / out of the pair. The pair only inspects `balanceOf` for the delta — naive "LP add" detection via `Mint` alone misses who funded it.
3. **Pair `Transfer(0x0, 0x0, 1000)` on the very first `mint`** is the permanent `MINIMUM_LIQUIDITY` lock. Exclude from "total LP minted" aggregates.
4. **The 30 bps swap fee never emits an event.** Enforced by `amountIn * 997 / 1000` inside `swap()`. To attribute fee revenue, derive `0.003 * amountIn_normalized` from `Swap`.
5. **Protocol fees (when `feeTo != 0`) are minted as LP, not transferred.** On `mint`/`burn` after a `k` increase, the pair mints fresh LP shares to `feeTo` worth ~1/6 of the swap fee — observable as Pair `Transfer(0x0, feeTo, …)` with no surrounding ERC-20 movement. ON for ETH/Base/Arbitrum/Optimism/BNB/Polygon; OFF only for Avalanche.
6. **Flash loans = `Swap` events with non-zero `amountIn` AND `amountOut` on the same side.** The pair callbacks `uniswapV2Call(sender, amount0, amount1, data)` on `to` between sending out and the K-check.
7. **`Swap.sender` is the Router** (or whatever contract called `swap()`), not the trader. For Universal-Router / 1inch / 0x flows, `tx.from` is the user but `Swap.sender` is the aggregator. Don't equate `Swap.sender` with the trader.
8. **`token0 < token1` is a factory invariant.** A pair where this doesn't hold is a fork, not Uniswap-Labs V2.
9. **Init code hash is identical on all 7 supported chains** (§11) — one derivation function works everywhere. NOT the case for V2 forks: PancakeSwap, SushiSwap, BiSwap each have chain-specific hashes.
10. **Router01 traffic still happens (Ethereum only).** ~1-3% of mainnet V2 swap volume routes through `0xf164…b92a` — legacy bots and old interfaces. Router01 does not exist on any other chain.
11. **Permit2 is NOT V2-native.** Universal Router / x402 / gasless flows use `Permit2.permitTransferFrom`; the V2 Router itself doesn't integrate Permit2.
12. **BSC has 18-decimal stablecoins.** USDC, USDT, and BUSD on BNB are 18 decimals (not 6). A `value / 1e6` shortcut over-counts by 10¹² on BSC.
13. **"USDC.e" symbol mismatch.** Bridged USDC on Optimism / Arbitrum / Polygon returns `symbol() = "USDC"` (no `.e` suffix). Distinguish by address, never symbol.
14. **WPOL ≠ WMATIC contract change.** Same contract (`0x0d50…1270`); only `symbol()` updated post-rebrand. Address / decimals unchanged.
15. **Factory addresses cluster across chains.** `0x8909…18eC6` lives on Base AND BNB; `0x9e5A…799C` on Avalanche AND Polygon. Always condition queries on `(chainId, address)`.

## 14. Verification & sources

How every constant in this doc was verified:

- **Function selectors and topic0 hashes:** computed locally as `keccak256(canonical signature)[0:4]` / full `keccak256` (pycryptodome `Crypto.Hash.keccak`). Canonical signatures taken from `Uniswap/v2-core` and `Uniswap/v2-periphery` master.
- **Address bytecode existence:** `eth_getCode` on each chain's public RPC returns non-empty bytecode for every listed contract. Factory + Router02 are byte-for-byte identical across chains (13 859 B and 17 891 B respectively, except Ethereum Router02 which is 21 943 B from an older solc).
- **Factory state:** `eth_call(factory, 0x574f2ba3)` for `allPairsLength()`, `0x017e7e58` for `feeTo`, `0x094b7415` for `feeToSetter` on each chain — values quoted live as of 2026-05-21.
- **Router wiring:** `eth_call(router, 0xc45a0155)` (`factory()`) and `0xad5c4648` (`WETH()`) on each chain returns the listed factory + wrapped-native addresses.
- **Pool init code hash:** reverse-derived from each chain's live `WrappedNative/USDC` V2 pair (returned by `getPair`) — CREATE2 round-trip under candidate hash `0x96e8ac42…48845f` matches on every chain (Ethereum `0xB4e16d…C9Dc`, Base `0x88a43B…bb9c`, BNB `0x57c68f…cfeb`, Avalanche `0x6239Ae…E391`, Arbitrum `0xF64dfe…366a`, Optimism `0x4C4364…7815`, Polygon `0x1F0c54…D868`).
- **Token decimals + symbols:** `decimals()` and `symbol()` read live per chain. BSC 18-decimal stablecoins, Polygon's `"WPOL"` post-rebrand symbol, and Arbitrum's `"USD₮0"` Tether-0 brand were all confirmed.

# Uniswap V3 — Topics, Selectors, Addresses (Ethereum, Base, Arbitrum, Optimism, Polygon, BSC, Avalanche)

**Status:** verified against live RPC on every listed chain and the canonical `Uniswap/v3-core`, `Uniswap/v3-periphery`, `Uniswap/v3-staker`, `Uniswap/universal-router` repos on 2026-05-21.
**Scope:** Uniswap Labs' V3 deployment on Ethereum (1), Base (8453), Arbitrum One (42161), Optimism (10), Polygon PoS (137), BNB Smart Chain (56), Avalanche C-Chain (43114). Most material is chain-agnostic; addresses are network-specific.

Uniswap V3 is **immutable** for all core swap/liquidity contracts. The only upgradeable piece is the NFT image renderer (`NonfungibleTokenPositionDescriptor`), which sits behind an OpenZeppelin `TransparentUpgradeableProxy` (EIP-1967). Everything else — Factory, Pool, NPM, SwapRouter(s), Quoter(s), V3Migrator, V3Staker, Universal Router — is deployed without proxies and cannot be upgraded in place.

Pools are deployed by the Factory via `CREATE2` with `salt = keccak256(abi.encode(token0, token1, fee))`. Token order is enforced lexicographically (`token0 < token1`). The pool init code hash is **chain-agnostic**: `0xe34f199b19b2b4f47f68442619d555527d244f78a3297ea89325f843f87b8b54`. A pool's address is therefore deterministic given the factory address and `(token0, token1, fee)`.

Fee tiers and their tick spacings (verified on-chain; `factory.feeAmountTickSpacing(fee)`):

| Fee (uint24) | Fee % | tickSpacing | Enabled on Ethereum | Enabled on Base | Typical use |
|--------------|-------|-------------|---------------------|-----------------|-------------|
| 100          | 0.01% | 1           | yes (Nov 2022+)     | yes             | stable↔stable (USDC/USDT) |
| 500          | 0.05% | 10          | yes                 | yes             | stable↔ETH, low-vol |
| 3000         | 0.30% | 60          | yes                 | yes             | default / blue-chip |
| 10000        | 1.00% | 200         | yes                 | yes             | exotic / long-tail |

### 8.4 Pool deployment (CREATE2, immutable per pair)

Pools are deployed by the factory's `createPool(tokenA, tokenB, fee)` flow:

1. Factory orders tokens: `(token0, token1) = tokenA < tokenB ? (tokenA, tokenB) : (tokenB, tokenA)`.
2. Factory writes `parameters` storage with `(factory, token0, token1, fee, tickSpacing)`.
3. Factory `CREATE2`s `UniswapV3Pool` with `salt = keccak256(abi.encode(token0, token1, fee))` and the pool constructor reads `parameters()` from the factory.
4. Factory clears `parameters` and emits `PoolCreated`.

The deployed pool is **immutable** (no proxy slot, no upgrade hook). Once created, a pool address is permanent. New fee tiers can only appear if governance calls `enableFeeAmount` on the factory.

## 9. Detection invariants & gotchas

These are the non-obvious things to lean on when indexing V3.

1. **`PoolCreated.token0 < token1` always**, regardless of how the caller passed them. The Factory sorts. Build your pool-discovery query around the canonical sorted pair, not user-supplied order.
2. **Pool address ≡ deterministic CREATE2** of `(factory, token0, token1, fee, init code hash)`. You can validate a claimed pool address purely off-chain — useful when accepting pool addresses from untrusted callers.
3. **`Swap.amount0` and `Swap.amount1` are signed.** Positive = into pool, negative = out of pool. `Swap.recipient` is **never** equal to `tx.origin` for router flows — it's typically the router itself, and then `Transfer` events redistribute. Don't infer payer from `recipient`.
4. **`Swap` does not emit `Transfer`** from the pool — instead, the swap callback pulls the input. The matching ERC-20 `Transfer` events are emitted by the input/output token contracts, not by the pool. To compute "the swap moved X token from A to B", join `Swap` with the pool's input/output token `Transfer` events in the same tx.
5. **NPM `Collect.amount0/amount1`** are the gross amounts withdrawn including both fees and unaccounted decreaseLiquidity remainder. To compute "fees earned", subtract any preceding `DecreaseLiquidity.amount0/amount1` in the same multicall.
6. **NPM `Burn` requires both `liquidity == 0` and `tokensOwed0/1 == 0`.** A common UX flow is multicall: `decreaseLiquidity → collect → burn`. Indexers should treat the trio as one logical "close position" event.
7. **`Swap` is emitted by the Pool, not the Router.** Filter `topic0 = SwapTopic` + restrict to known pool addresses (derived from `PoolCreated` history) to avoid false positives from V2/Sushi/etc.
8. **`Initialize` fires exactly once per pool**, before any swaps. It carries the **initial** `sqrtPriceX96`. To bootstrap a TWAP index, capture the `Initialize` block; before that block the pool is empty and `slot0()` is `(0, 0, …)`.
9. **`SetFeeProtocol` is governance-only** and currently `0` on every Ethereum pool (no protocol fee turned on at the time of writing). Indexing protocol-fee accrual requires watching this event per pool.
10. **TWAP cardinality must be grown explicitly** (`increaseObservationCardinalityNext`). A freshly created pool has cardinality 1 — any TWAP older than the most recent swap will return the latest observation. Don't trust TWAP from low-cardinality pools.
11. **Pool init code hash is the same across all Uniswap V3 chains** (Ethereum, Polygon, Arbitrum, Optimism, Base, BNB, Avalanche, …). Only the factory address differs. This is the chain-portability cornerstone.
12. **Universal Router commands can mix V2, V3, V4, NFT, and Permit2 actions in one tx.** A single tx may emit V2 `Swap`, V3 `Swap`, ERC-721 `Transfer`, Permit2 `UnorderedNonceInvalidation`, and ERC-20 `Transfer` all from different contracts. To attribute a Uniswap V3 swap to UR, filter `topic0 = V3 Swap` + `pool.address ∈ V3 pools` and then look for `tx.to ∈ {UR set}`.
13. **`msg.sender` in `Swap.sender` is the caller of `pool.swap()`** — usually a router, not the actual user. The user is somewhere upstream (in the input-token `Transfer.from`).
14. **Pool tokens of zero value / fee-on-transfer tokens partially break V3.** V3 assumes `amount-in pre-transfer == amount-in post-transfer`. Fee-on-transfer tokens (e.g., legacy SAFEMOON-style) will cause swap reverts because the callback's repayment mismatches. Some pools that contain such tokens exist but are perpetually broken; you'll see `PoolCreated` but no `Swap`.
15. **NFT permit (`NPM.permit`)** uses ERC-4494 (similar to ERC-2612 but for ERC-721). The selector is `0x7ac2ff7b`. Don't confuse with `selfPermit` (which is for ERC-20 tokens used as inputs to swap routers).

## 11. Verification & sources

How every constant in this doc was verified:

- **Function selectors and topic0 hashes:** computed locally as `keccak256(canonical signature)[0:4]` / full `keccak256` (pycryptodome `Crypto.Hash.keccak`). Canonical signatures taken from the official `Uniswap/v3-core`, `Uniswap/v3-periphery`, `Uniswap/v3-staker`, `Uniswap/universal-router` repos.
- **Address bytecode existence:** `eth_getCode` on `https://ethereum-rpc.publicnode.com`, `https://base-rpc.publicnode.com`, `https://arbitrum-one-rpc.publicnode.com`, `https://optimism-rpc.publicnode.com`, `https://polygon-bor-rpc.publicnode.com`, `https://bsc-rpc.publicnode.com`, `https://avalanche-c-chain-rpc.publicnode.com` returns non-empty bytecode for every listed contract on its respective chain.
- **EIP-1967 impl pin:** `eth_getStorageAt(proxy, 0x360894…bbc)` on Ethereum / Arbitrum / Optimism / Polygon returns `0x91ae842A5Ffd8d12023116943e72A606179294f3` and on Base returns `0x4f225937EDc33EFD6109c4ceF7b560B2D6401009`, matching the listed impls byte-for-byte.
- **Factory owner:** `eth_call(factory, 0x8da5cb5b)` returns `0xf2371551fe3937dB7c750f4dfaBE5C2fFFDcBF5A` (Ethereum), `0xabEa76658B205696D49B5F91b2A03536cb8a3Be1` (Base), `0xFf7ad5da31fEcDc678796c88b05926Db896B0699` (Arbitrum), `0xeC23CF5a1db3dCc6595385d28B2A4D9b52503Be4` (Optimism), `0x8A1B966AC46F42275860F905dbc75efbfDc12374` (Polygon), `0x3f07F08b45912dCD6691c5B9412975D5113B2910` (BSC), `0xeB0bcF27d1FB4b25e708fbb815c421AeB51EA9Fc` (Avalanche).
- **Fee tier enablement:** `eth_call(factory, 0x22afcccb || feeAsUint256)` returns `1, 10, 60, 200` for fees `100, 500, 3000, 10000` on every chain.
- **Pool init code hash:** taken from `v3-periphery/contracts/libraries/PoolAddress.sol` constant; cross-checked by deriving a known pool address (e.g., USDC/WETH 0.05% on Ethereum = `0x88e6A0c2dDD26FEEb64F039a2c41296FcB3f5640`) from the formula and confirming the on-chain match. Same hash works on all 7 chains.
- **Per-chain core bytecode equivalence:** SwapRouter02 bytecode size matches across Ethereum, Arbitrum, Optimism, Polygon, BSC, and Avalanche (`24497 B` runtime) — same compiled output redeployed. Base is smaller (`17891 B`) due to a different solc/optimizer setting.
- **OP missing V3Migrator** and **BSC missing V3Migrator** confirmed by `eth_getCode` returning `0x` at `0xA5644E…` (OP) and `0x32CDA1…` (BSC).

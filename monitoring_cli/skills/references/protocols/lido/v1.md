# Lido V1 — Topics, Selectors, Addresses (Ethereum only) — LEGACY / PRE-SHAPELLA

**Status:** V1-era event topics verified against live historic `eth_getLogs` on the Lido stETH proxy and the old `LidoOracle` (now `LegacyOracle`, `0x442af7…43fb`) in a 2021 window; selectors computed via keccak-256 (sanity-checked against ERC-20). Verified 2026-05-29.
**Scope:** "Lido V1" = the **original Lido on Ethereum, Dec 2020 → May 2023 (pre-Shapella).** Ethereum mainnet **only** (no L2 / sidechain Lido existed in this era — see [l2.md](l2.md) for later bridged wstETH). The protocol was **upgraded in place** to V2 (Shapella) then V3 (stVaults): the **same proxy addresses** carry today's V2/V3 logic, so this file is for **historic indexing** of pre-May-2023 blocks. For the live protocol use [v2.md](v2.md) and [v3.md](v3.md).

**What V1 was.** Stake-only liquid staking: `submit` ETH → mint rebasing **stETH**; ETH forwarded to the beacon deposit contract via curated node operators. **No withdrawals** (locked until Shapella), **no Staking Router** (a single hard-wired `NodeOperatorsRegistry`), and a **push-model oracle** (`LidoOracle`): a quorum of members called `reportBeacon`, the oracle pushed the new beacon balance into Lido (`pushBeacon`, later `handleOracleReport`), and stETH rebased. EL-rewards (`LidoExecutionLayerRewardsVault`) were added mid-2021. The rebase fan-out event was the oracle's **`PostTotalShares`** (V2 replaced it with `TokenRebased` on the Lido contract).

## 4. Proxies

Same Aragon model as today: **Lido, NodeOperatorsRegistry, LegacyOracle and the Aragon apps are `AppProxyUpgradeable`** (EIP-1967 slot **zero**; impl via `Kernel.getApp`). V1 had **no OssifiableProxy contracts** — the EIP-1967-proxied stack (LidoLocator, StakingRouter, WithdrawalQueue, AccountingOracle, VEBO) is a V2 introduction. To inspect a V1-era impl, read the historic state of the Aragon Kernel at that block.

## 5. Detection invariants & gotchas (historic indexing)

1. **No withdrawals before block ~17,264,000 (Shapella, May 2023).** No `WithdrawalQueueERC721`, no `WithdrawalRequested`/`Claimed`. stETH was one-way until then.
2. **Rebase = oracle `PostTotalShares`, not `TokenRebased`.** Key V1 rebase analytics on the LegacyOracle `0x442af7…` and `PostTotalShares` (`0xdafd48d1…`).
3. **Single operator registry, no StakingRouter.** Deposits went `Lido.depositBufferedEther` → NOR → deposit contract; there's no `StakingRouterETHDeposited`.
4. **Oracle is push-model.** `reportBeacon` (members) → `Completed`/`PostTotalShares`; no `HashConsensus`, no `AccountingOracle`, no `submitReportData`.
5. **EL rewards appear only after Sept 2021** (`ELRewardsReceived`, `LidoELRewardsVaultSet`).
6. **`TransferShares`/`SharesBurnt` arrive mid-V1** — earliest blocks lack them; rely on `Transfer` + share-rate reconstruction there.
7. **Same proxy addresses as V2/V3** — never infer the version from the address; infer it from the block height / live impl.

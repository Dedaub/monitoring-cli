# Lido on L2s & Sidechains — Topics, Selectors, Addresses (Base, Arbitrum, Optimism, BNB, Polygon, Avalanche)

**Status:** verified 2026-05-29 via `eth_getCode` / `name()` / `symbol()` / EIP-1967 slot reads on per-chain public RPCs, against `docs.lido.fi/deployed-contracts` and `lidofinance/lido-l2`. Token/event signatures are chain-agnostic; addresses are network-specific.
**Scope:** Lido's **non-Ethereum** footprint. The core staking protocol lives only on Ethereum ([v2.md](v2.md) / [v3.md](v3.md)); on every other chain Lido is the **bridged `wstETH` token** (± a rebasable L2 `stETH` and the bridge/rate-oracle contracts), plus the now-**sunset `stMATIC`** product for Polygon. There is **no staking, no withdrawals, no oracle** on these chains — only bridged tokens.

## Official-status summary — READ FIRST

| Chain | ID | Official Lido bridge? | Mechanism |
|-------|----|-----------------------|-----------|
| Optimism | 10 | **YES** | Native Lido OP-Stack bridge (`L1/L2ERC20ExtendedTokensBridge`) — wstETH **+ rebasable stETH + TokenRateOracle** |
| Base | 8453 | **YES** | Native Lido OP-Stack bridge (`L1/L2ERC20TokenBridge`) — **wstETH only** |
| Arbitrum | 42161 | **YES** | Native Lido Arbitrum gateway (`L1/L2ERC20TokenGateway`) — **wstETH only** |
| BNB Chain | 56 | **YES** (current) | Wormhole × Axelar **NTT** (Native Token Transfer), DAO-owned — **wstETH only** |
| Polygon PoS | 137 | **NO** | wstETH = Polygon's **canonical PoS bridge** (UChildERC20). Separately, **stMATIC** = the **sunset** "Lido on Polygon" product |
| Avalanche | 43114 | **NO** | No official Lido deployment — only a third-party **LayerZero OFT** wstETH |

> **wstETH on L2 does not rebase** (it's the same non-rebasing token as L1). Only Optimism has an official **rebasable `stETH`** (backed by bridged wstETH + a `TokenRateOracle` that mirrors L1 `wstETH.stEthPerToken()`). Elsewhere, convert wstETH→stETH value off the L1 rate or the chain's TokenRateOracle.

## 5. Detection invariants & gotchas

1. **No staking/withdrawals/oracle off Ethereum.** These chains only mint/move bridged tokens. Stake/unstake events live on Ethereum ([v2.md](v2.md)).
2. **L2 wstETH does not rebase** — only Optimism's `stETH` (`0x76A5…4138`) rebases (via TokenRateOracle). Everywhere else convert via the L1/oracle rate (`stEthPerToken`).
3. **Same address ≠ same chain.** wstETH addresses differ per chain; the LayerZero OFT `0x2Bbbdf…` is the only one reused (BNB + Avalanche) — and it's **unofficial**.
4. **BNB has two wstETH tokens.** Canonical = NTT `0x26c5…`; the LayerZero OFT `0x2Bbbdf…` is a separate, non-canonical token. Don't conflate balances.
5. **Polygon & Avalanche are not official Lido bridges.** Polygon wstETH is Polygon's canonical PoS bridge; Avalanche wstETH is a third-party OFT. State this in any report.
6. **stMATIC is a different asset** (staked MATIC), now sunset — don't mix it with wstETH staking analytics.
7. **Polygon wstETH proxy is not EIP-1967** — reading slot `0x3608…` returns empty; use Polygon's proxy slot or the docs impl.

# SushiSwap V3 — Compressed Deployment Reference (Ethereum, Base, BSC, Avalanche, Arbitrum, Optimism, Polygon PoS)

**Status:** topic0/selectors via `cast keccak`/`cast sig` (verified); addresses from the `sushi-labs/sushi` SDK + on-chain re-verified with `cast` vs `publicnode` RPCs (2026-05) — every NFPM `symbol()` returned "SUSHI-V3-POS" and `factory()` captured authoritatively.
**Scope:** SushiSwap **V3** (concentrated liquidity — a Uniswap V3 fork). Other versions: [`v2.md`](v2.md) (classic AMM), [`trident.md`](trident.md) (BentoBox AMM). **Protocol-wide pieces** (SUSHI token, MasterChef/MiniChef staking, RedSnwapper/RouteProcessor router) are documented in [`v2.md`](v2.md) — V3 swaps also route via RedSnwapper.
**Key fact:** V3 is a Uniswap V3 fork → **topic0s and selectors are IDENTICAL to Uniswap V3** (`uniswap/v3.md`); the pool init code hash is even the **same** as Uniswap's (`0xe34f199b…`, Sushi forked the pool verbatim). Only the factory/NFPM addresses differ. **V3 is deployed on all 7 chains.**

## Proxies

- **V3: immutable core** (Uniswap V3 fork). Factory / Pool / NFPM are not upgradeable; pools are CREATE2-deployed with init hash `0xe34f199b…` (same as Uniswap V3). As with Uniswap, the NFPM's `tokenDescriptor` (SVG renderer) may sit behind an EIP-1967 `TransparentUpgradeableProxy` — verify per deployment.
- Router (RedSnwapper) and staking proxies: see [`v2.md`](v2.md). Trident's clone-proxy model: see [`trident.md`](trident.md).

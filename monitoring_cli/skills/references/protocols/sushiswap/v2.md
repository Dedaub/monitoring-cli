Other versions: [`v3.md`](v3.md) (concentrated liquidity), [`trident.md`](trident.md) (BentoBox AMM, deprecated). **This file also holds the protocol-wide pieces** — SUSHI token, MasterChef/MiniChef staking, and the RedSnwapper/RouteProcessor router — which serve all versions; v3/trident reference here.
**Key fact:** V2 is a Uniswap V2 fork → **topic0s and selectors are IDENTICAL to Uniswap V2** (`uniswap/v2.md`); only the factory addresses and the **pair init code hash** (`0xe18a34eb…`, Sushi-specific) differ. **Classic V2 is deployed on Ethereum, Arbitrum, Polygon, BSC, Avalanche — NOT on Base/Optimism** (those chains are V3 + RedSnwapper era).

## Proxies

- **V2: NO proxies.** Factory, Router, Pair are immutable. Pairs are CREATE2-deployed by the factory with salt `keccak256(token0,token1)` and **Sushi's own init code hash** `0xe18a34eb…` (≠ Uniswap's `0x96e8ac42…`) — derive pair addresses off-chain per chain by swapping in that chain's factory address.
- **MasterChef / MasterChefV2 / MiniChefV2: not proxies** (immutable). **SUSHI token: not a proxy. RedSnwapper / RouteProcessor: not proxies** (new router versions are fresh deploys at new addresses).
- For V3 / Trident proxy patterns see [`v3.md`](v3.md) / [`trident.md`](trident.md).

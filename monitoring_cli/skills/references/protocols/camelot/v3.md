# Camelot V3 — Compressed Deployment Reference (Arbitrum / Algebra V1 fork)

**Status:** topic0/selectors computed via keccak-256 from canonical Algebra V1 source (`cryptoalgebra/AlgebraV1`); Swap/Mint/Burn/Collect/IncreaseLiquidity **confirmed against live `eth_getLogs`** on the WETH/USDC pool + NFPM on Arbitrum (2026-06-09, publicnode). All core addresses verified via `eth_getCode`/`eth_call`. `eth_getCode = 0x` confirmed on all 6 non-Arbitrum chains.
**Scope:** Camelot **V3** — a concentrated-liquidity AMM built on a **Camelot-forked Algebra V1** engine, deployed exclusively on **Arbitrum One (42161)**. Camelot adds a custom per-swap event and a modified `globalState()` (8-field vs Algebra V1 canonical 7-field). Other versions: [`v2.md`](v2.md) (standard Camelot legacy AMM).
**Key facts:**
- **Arbitrum-only** — `eth_getCode = 0x` on ETH/Base/BNB/AVAX/OP/Polygon.
- **Algebra ≠ Uniswap V3.** Dynamic per-block fee (no fee tiers) → `createPool(tokenA,tokenB)` has **no fee param**; factory event is `Pool(token0,token1,pool)` (no `uint24 fee`). Pools are CREATE2-deployed by a separate **PoolDeployer**. Pool init code hash is **Camelot-specific** (differs from QuickSwap V3 / canonical Algebra).
- **`Swap` topic0 is identical to Uniswap V3** (same 7-param ABI). To disambiguate: look for the `Fee(uint16)` event (`0x598b9f04…`, Algebra-only), or trace the pool back to the Camelot factory via the `Pool` event (`0x91ccaa7a…`), or check NFPM `symbol()="ALGB-POS"`.
- **`Mint` topic0 differs from Uniswap V3** — the Algebra V1 `Mint` event has 7 params (sender + 3 indexed: owner, bottomTick, topTick + 3 data). The commonly cited "extra `uint256 index`" for Algebra Mint is **incorrect** for V1; the actual Algebra V1 Mint has no index field and produces `0x7a53080b…`, not the Uniswap V3 `0xd78218c0…`.
- **Camelot-specific event** `0x8a89de70…` fires on every swap (LOG1, 2 uint256 data values = fee applied to each side). Not in upstream Algebra V1 source — embedded directly in Camelot's forked pool bytecode.
- **NFPM name/symbol:** `name()="Algebra Positions NFT-V1"`, `symbol()="ALGB-POS"`. Identical branding to other Algebra V1 deployments (QuickSwap V3, etc.).
- **No farming active** — `factory.farmingAddress() = 0x0000…0000` (not set). No FarmingCenter deployed or linked.

## Detection invariants & gotchas

1. **`Swap` topic0 `0xc42079…` is shared with Uniswap V3.** Discriminate via: `Fee(uint16)` `0x598b9f04…` (Algebra-only, fires frequently), factory `Pool` event `0x91ccaa7a…` (3 params, no fee), or NFPM 6-arg `IncreaseLiquidity` `0x8a82de7f…`, or pool `symbol()="ALGB-POS"`.
2. **`Mint` topic0 `0x7a53080b…` == Uniswap V3 pool Mint** (identical — Algebra V1 uses the same 7-param Mint with no extra index). The "extra `uint256 index`" in Algebra Mint seen elsewhere (e.g. Pharaoh/Ramses forks) is specific to a different Algebra fork, not V1. Use `Fee(uint16)` or factory `Pool` event to distinguish Camelot pools from UniV3.
3. **Camelot-specific swap event `0x8a89de70…`.** Fires as LOG1 (1 topic = the hash itself, no indexed params) with data=(fee0, fee1) in uint256 on every swap. Hardcoded in the forked pool bytecode. Not present on QuickSwap V3 or canonical Algebra V1 pools.
4. **Dynamic fee.** No fixed fee tier. Read live fee from `pool.globalState()` field 3 (uint16, units of 1e-6, so 100 = 0.01%). Fee changes emit `Fee(uint16)`. Do NOT assume 0.3%.
5. **One pool per token pair.** `poolByPair(a,b)` returns exactly ONE address. Pool CREATE2 salt = `keccak256(abi.encode(token0, token1))` with `token0 < token1`, deployed by PoolDeployer (not factory). Init code hash is **Camelot-specific** (not `0x6ec6c9c8…`).
6. **`Swap.amount0/amount1` are signed int256** (negative = out of pool, positive = into pool). `price` field is sqrtPriceX96 (Q64.96).
7. **`globalState()` returns 8 ABI-decoded slots** on Camelot V3 (canonical Algebra V1 returns 7). Slot 4 (value ~16766 on WETH/USDC) is a Camelot-specific addition — do not assume the 5th return value is `communityFeeToken0`.
8. **No farming integration via factory.** `factory.farmingAddress()` is zero. The `Incentive` event (`0x915c53…`) and `LiquidityCooldown` event (`0xb5e516…`) exist in the ABI but are not actively used.

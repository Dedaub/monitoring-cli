# DODO V1 (classic PMM) — Topics, Selectors, Addresses (Ethereum, BNB, Avalanche, Arbitrum, Optimism, Polygon)

**Status:** verified against live RPC on every listed chain and the canonical `DODOEX/dodo-smart-contract` (V1) + `DODOEX/contractV2` (V1 helpers) repos on 2026-06-02.
**Scope:** the original DODO **Proactive Market Maker (PMM)** generation — the `DODOZoo` registry/factory, the classic `DODO` pair contract (one isolated base/quote pool per pair), the read-only quoting helpers (`DODOSellHelper`/`DODO V1 PoolHelper`, `DODOV1PmmHelper`), and the V1 liquidity-mining contract (`DODOMine`, on Arbitrum). Topics + selectors are **chain-agnostic** (computed from the canonical event/function signatures); addresses are network-specific. **DODO V1 is NOT deployed on Base (8453)** — `base-config.js` carries no `DODOZoo`, and the Ethereum `DODOZoo` address returns `0x` from `eth_getCode` on Base. Only the chain-shared read-only V1 helpers (`DODOSellHelper`, `DODOV1PmmHelper`) exist on Base.

DODO V1 pools are **not deployed individually** and are **not EIP-1967 upgradeable proxies.** The `DODOZoo` registry holds a single `_DODO_LOGIC_` implementation and a `_CLONE_FACTORY_`; `breedDODO(...)` asks the CloneFactory to mint an **EIP-1167 minimal-proxy clone** of that logic, calls `init(...)` on the clone, registers it, and emits `DODOBirth(newBorn, baseToken, quoteToken)`. Every classic pair is therefore a ~45-byte EIP-1167 clone (`363d3d373d3d3d363d73<impl>5af43d82803e903d91602b57fd5bf3`) delegating all calls to a per-chain shared implementation — its own EIP-1967 impl slot is empty, and it is *not* individually verified on explorers. **Discover pairs from `DODOZoo.DODOBirth` logs (and `getDODOs()`), not from a hard-coded list** — only a handful of named pairs are listed here. The `DODOZoo` registry itself is an immutable `Ownable` contract (EIP-1967 impl slot `0x0`); upgrading `_DODO_LOGIC_` only changes which implementation *future* clones point at — already-bred pairs keep delegating to the implementation that was current when they were minted (which is why the long-lived Ethereum pairs report `version() == 100` while the current repo source is `101`).

A classic DODO pair is a **single-sided, oracle-priced, two-token pool**: it has a base token and a quote token (`_BASE_TOKEN_`/`_QUOTE_TOKEN_`), an external price oracle (`_ORACLE_`), and two separate LP share tokens (`_BASE_CAPITAL_TOKEN_`/`_QUOTE_CAPITAL_TOKEN_`, each a `DODOLpToken` "DLP" ERC-20). Traders call `sellBaseToken`/`buyBaseToken`; LPs `depositBase`/`depositQuote` and `withdrawBase`/`withdrawQuote`. **There is no `Swap` event and no `Mint`/`Burn` pool event** — trades emit `SellBaseToken`/`BuyBaseToken`, liquidity emits `Deposit`/`Withdraw` (with an `isBaseToken` bool, *not* the EIP-4626 shape), and accrued LP fees emit `Donate`. This is structurally different from DODO V2 (DVM/DPP/DSP factory clones with a `DODOSwap` event) — keep V1 and V2 detection separate.

## 12. Detection invariants & gotchas

1. **Discover pairs from `DODOZoo.DODOBirth` logs (topic0 `0x5c428a2e…`), never from a static list.** All three args are unindexed in `data`: `[newBorn, baseToken, quoteToken]`. `getDODOs()` gives the current full set; named pairs in §3–§8 are a subset.
2. **A pair is an EIP-1167 clone, not a verified contract and not an EIP-1967 proxy.** Block explorers usually show no source for the clone address; its EIP-1967 impl slot is empty. Don't conclude "unverified/empty" — decode the clone bytecode to find the shared impl.
3. **Three different events are named `Deposit`** with different topic0s: V1 pair `Deposit(address,address,bool,uint256,uint256)` = `0x18081cde…`; V1 `DODOMine` `Deposit(address,uint256,uint256)` = `0x90890809…`; and the unrelated EIP-4626 `Deposit`. The same applies to `Withdraw`. **Always key on `(chainId, address, topic0)`** — never topic0 alone.
4. **V1 has no `Swap` event.** Trades are `SellBaseToken`/`BuyBaseToken` (each with the trader as the single indexed topic1). To capture all V1 trading on a pair, watch **both** topics. V2 (DVM/DPP/DSP) uses a single `DODOSwap` event — do not conflate.
5. **`Donate` fires on (almost) every trade** — it credits the LP fee to the pool target. High-volume pairs emit it far more often than the trade events themselves (it also fires for withdrawal penalties). Useful as a liveness signal, noisy as a trade proxy.
6. **A pair holds two LP tokens, not one.** `_BASE_CAPITAL_TOKEN_()` and `_QUOTE_CAPITAL_TOKEN_()` are separate `DODOLpToken` ("DLP") ERC-20s; base LPs and quote LPs are independent. `Transfer`/`Mint`/`Burn` on those token addresses (not the pair) reflect LP-share movements.
7. **`version()` of long-lived pairs returns `100`, not the repo's `101`.** The clone impl that was current at breed time is frozen into the pair. Don't assert `version()==101`.
8. **`DODOZoo` is deployed but empty on Avalanche and Optimism** (`getDODOs()==0`). Finding the Zoo there does **not** mean V1 pools exist — confirm with `getDODOs()` length before indexing pairs.
9. **DODO V1 is absent on Base.** The Zoo returns `0x` on Base; only the read-only helpers exist. Any "DODO V1 pool on Base" is a misclassification (likely a V2 DVM/DPP/DSP clone).
10. **The "DODO V1 PoolHelper" registry entry == `DODOSellHelper`** (same address & bytecode) on Ethereum, and is the per-chain `DODOSellHelper`-family helper elsewhere. It's a read-only quoter — it emits no events and holds no funds.
11. **Pairs use an external oracle for pricing** (`_ORACLE_`/`getOraclePrice`). PMM math (`_K_`, R-status, target reserves) means the marginal price is oracle-anchored, not constant-product. Reserve-ratio reasoning from Uniswap does not transfer.
12. **The V1 "DODO V1 Mining" (Arbitrum, `0xE3C1…fDe3e`) is the classic `DODOMine`** (MasterChef-style), distinct from V2's `ERC20MineV2`/`ERC20MineV3` and `DODOMineV3Proxy`. Its `Deposit`/`Withdraw` carry `(user, indexed pid, amount)` and also emit `Claim` on harvest.
13. **No standalone V1 trade proxy (`DODOEthProxy`/`DODOV1Proxy`) is deployed in this registry on any of the 7 chains.** The source exists in the V1 repo (§1.8/§2.6) but the historical mainnet instance returns `0x`; live V1 swaps route through the V2-era `DODOV1Adapter` (called by `DODORouteProxy`/`DODOV2Proxy`). If you see V1 pair trades, the `tx.to` is typically a V2 route proxy, not a V1 proxy.
14. **Ownership is 2-step and uses non-standard names.** `OwnershipTransferPrepared` (= the "started" event) then `OwnershipTransferred` on `claimOwnership()`. The owner storage var is `_OWNER_()` (`0x16048bc4`), not OpenZeppelin's `owner()` (`0x8da5cb5b`) — the latter **reverts** on V1 contracts.

# Euler V2 — Topics, Selectors, Addresses (Ethereum, Base, BNB, Avalanche, Arbitrum, Polygon)

**Status:** verified 2026-05-29. All addresses parsed from the canonical `euler-xyz/euler-interfaces` repo (`addresses/<chainId>/*.json` — Euler's equivalent of `bgd-labs/aave-address-book`). Event topic0s and selectors computed locally with keccak-256 from that repo's `abis/*.json`. Addresses, `factory.implementation()`, the EVault proxy pattern, and the `EVC.CallWithContext` / `EVault.Deposit` topics verified against public RPCs on each chain.
**Scope:** Euler V2's deployment on Ethereum (1), Base (8453), BNB Smart Chain (56), Avalanche C-Chain (43114), Arbitrum One (42161), Polygon PoS (137). **Optimism (10) is NOT an official Euler V2 deployment** (absent from `euler-interfaces`). Euler V1 is a separate, dormant Ethereum-only deployment — see [v1.md](v1.md).

## Architecture — EVC + EVK (read this first)

Euler V2 is a **construction kit**, not a monolith. Two foundations plus periphery:

1. **EVC — Ethereum Vault Connector** (one singleton per chain): the authorization & cross-vault layer. It mediates *every* state-changing user interaction with vaults: it tracks an account's enabled **collaterals** and **controller** (the borrow vault), defers liquidity/vault status checks within a `batch`, supports sub-accounts (256 per owner, `owner XOR id` in low byte), operators, permits, and "call/control" routing. Vaults trust the EVC for `msg.sender` authentication. **Most user txs flow through `EVC.batch()` / `EVC.call()`**, so `EVC.CallWithContext` is the master trace event.
2. **EVK — Euler Vault Kit**: the **EVault**, an ERC-4626 lending vault. Vaults are deployed by **`GenericFactory`** as 366-byte beacon proxies. The factory **is the beacon** (`factory.implementation()` returns the current `eVaultImplementation`). Each EVault implementation is itself a **module dispatcher** that `delegatecall`s eight baked-in module-logic contracts (Initialize, Token, Vault, Borrowing, Liquidation, RiskManager, BalanceForwarder, Governance) — read them via `vault.MODULE_*()`.

Periphery (all per-chain): **EulerRouter** (oracle aggregator), IRM factories (kink / adaptive-curve), **Perspectives** (on-chain vault-property verifiers / registries), **EulerEarn** (ERC-4626 yield aggregator over vaults; MetaMorpho-style governance), **EulerSwap** (just-in-time AMM that borrows from vaults; Uniswap-V4-hook compatible), FeeFlowController (dutch-auction fee buyback), Swapper/SwapVerifier, and Lens read-only helpers.

Per-vault addresses are **not** in any static list — discover them from `GenericFactory.ProxyCreated` (and `EVaultCreated` from the Initialize module), or `GenericFactory.getProxyListSlice(0, len)`.

## 6. Proxies & upgradeability

- **EVault proxies are NOT EIP-1967.** They are 366-byte minimal proxies deployed by `GenericFactory` with **trailing immutable data** (asset, oracle, unitOfAccount appended after the runtime bytecode). The EIP-1967 implementation slot reads **`0x0`** (verified). To find a vault's implementation, call `GenericFactory.implementation()` (beacon) — not a storage slot.
  - `upgradeable=true` vaults delegatecall to `factory.implementation()` (the beacon) → a `SetImplementation` event upgrades **all** of them at once.
  - `upgradeable=false` vaults bake the implementation in immutably (no upgrade path).
- **The EVault implementation is itself a module dispatcher** — it `delegatecall`s the eight `MODULE_*` logic contracts (§3.1). Internal calls to those module addresses appear in traces but never as top-level txs.
- **EulerEarn** vaults are deployed by `EulerEarnFactory`; **EulerSwap** pools by `EulerSwapV{1,2}Factory` (deterministic `CREATE2`, address derivable via `computePoolAddress`). EulerSwap pools are also Uniswap-V4 hooks.
- **EulerRouter** is a non-proxy governable singleton; reconfigured via `ConfigSet` / `govSetConfig`.
- Vault governance can be **revoked** (set governor to `0x0`) to make a vault immutable/"finalized" — watch `GovSetGovernorAdmin(0x0)`.

## 7. Detection invariants & gotchas

- **`EVC.CallWithContext` (`0x6e9738e5…`) is the master trace event** — almost every user action is a routed call/batch; the `targetContract` + `selector` fields tell you which vault and which op. Live-verified: 3,413 hits in a 30k-block Ethereum window.
- **`Deposit`/`Withdraw` topic0s are shared between EVault and EulerEarn** — disambiguate by log `address` (is it in `GenericFactory.proxyList` vs `EulerEarnFactory`?).
- **`msg.sender` on a vault is usually the EVC**, not the user. The real account is `EVC.getCurrentOnBehalfOfAccount()` / the `onBehalfOfAccount` field of `CallWithContext`. Don't attribute flows to the EVC address.
- **Sub-accounts:** an owner controls 256 accounts sharing the upper 19 bytes (`bytes19 addressPrefix`); the low byte is the sub-account id. `haveCommonOwner(a,b)` and `getAddressPrefix` collapse them.
- **`VaultStatus` is emitted on every state-changing vault op** — a cheap per-vault heartbeat for totalBorrows/cash/rate snapshots without decoding each action.
- **Discover vaults exclusively from `GenericFactory.ProxyCreated`** (or `getProxyListSlice`). There is no canonical vault list; anyone can deploy a vault. Use **perspectives** (`governedPerspective`, `evkFactoryPerspective`) to filter "official"/verified vaults from arbitrary ones.
- **Liquidations are per-vault** (`EVault.Liquidate`), unlike V1's single-contract `Liquidation`. `repayAssets`/`yieldBalance` are in the vault's asset / collateral-share units respectively.
- **Caps are exponent-encoded `uint16`** (not raw amounts): decode via `(10**(cap & 63) * (cap >> 6)) / 100`.
- **Optimism (10) has no official Euler V2** — ignore any "Euler" contracts there as third-party/unofficial.

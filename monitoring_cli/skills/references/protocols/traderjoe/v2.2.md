### LBHooksManager (deploys/registers hook contracts; deterministic `0x3A8E1Be9…`, EIP-1967 proxy)
```
Hooks are the V2.2 replacement for the old MasterChef farm model: liquidity rewards now attach to a pair as a hook.
LBHooksManager deploys typed hooks — LBHooksBaseRewarder / LBHooksExtraRewarder (LP reward hooks),
LBHooksMCRewarder, plus limit-order hooks (repo lfj-gg/joe-v2-periphery). Each hook instance emits its own
reward events. Discover hook deployments via LBHooksManager + the pair's HooksParametersSet log.
Hooks parameters are packed into bytes32 = (flags in the high bits | hook contract address in the low 160 bits),
so a non-zero getLBHooksParameters() reveals both that a hook is attached and its address.
```

## Cross-chain summary
- **LB 2.2 lives on Avalanche + Arbitrum only.** Avalanche is overwhelmingly dominant (16,288 pairs). The `0xb43120c4…` factory is **empty on BSC, Ethereum, Base, Optimism, Polygon**.
- **Autopools / Limit Orders / Aggregator** are deterministic on Avax/Arb (Vault Factory + APTFarm + LimitOrderManager also present on BSC), and route across **all** LB versions + V1.

## Proxies
- **Factory / Router / Quoter / Joe Aggregator / RouterLogic / ForwarderLogic: immutable.**
- **LBPair instances: clone-with-immutable-args** (97-byte clone → impl `0x7a5b4e30…`), same pattern as 2.1; watch `LBPairImplementationSet` `0x900d0e3d…` for future-pair impl changes.
- **Upgradeable (EIP-1967 proxies — watch `Upgraded`):** `LBHooksManager` (`0x3A8E1Be9…` → `0xc2a397a1…`), `LimitOrder` (`0xDbeeB3FB…` → `0x525a1882…`), Autopool `Vault Factory` (`0xA3D8…` → `0x7f0dff2f…`).
- **Hooks** attach extra logic to a pair via the packed `hooksParameters` `bytes32`. A non-zero `getLBHooksParameters()` (and a `HooksParametersSet` log) means a hook contract is intercepting that pair's mint/burn/swap.

## Detection invariants & gotchas
1. **2.2 is topic0-indistinguishable from 2.1** except for `HooksParametersSet` `0x4d24d81a…`. To separate the two, key on the **factory/pair address** (2.2 factory `0xb43120c4…` vs 2.1 `0x8e42f2F4…`/ETH `0xDC8d77b6…`).
2. **2.2 is the version to watch for current volume** — 16,288 Avax pairs and the live `Swap` flow (`0xad7d6f97…`). 2.0/2.1 are largely dormant.
3. **Hooks change pair behavior** — a pair with non-zero hooks can levy extra fees / gate actions; `HooksParametersSet` is the on-chain signal that a pair's hook was (re)pointed.
4. The **Joe Aggregator (`0x45A62B…`) is where most user swaps now enter** — it fans out to LB pools; the pool-level `Swap` topic0 is still `0xad7d6f97…`, so pool monitors catch the fills regardless of entry path.

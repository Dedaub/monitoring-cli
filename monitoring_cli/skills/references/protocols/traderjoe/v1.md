# Trader Joe V1 — Compressed Reference (Avalanche, Arbitrum, BNB Chain) + protocol-wide token/staking

**Status:** topic0/selectors computed locally (keccak) from `traderjoe-xyz/joe-core` and re-confirmed against `uniswap/v2.md`; addresses on-chain verified via `eth_call`/`eth_getCode`/`eth_getStorageAt` vs publicnode (2026-06-05).
**Scope:** **Joe V1** — the classic constant-product AMM (a **Uniswap V2 fork**) — **plus the protocol-wide token & staking layer** (JOE/xJOE/veJOE/rJOE, `MasterChefJoe V2/V3`, `BoostedMasterChefJoe`, `StableJoeStaking`=sJOE, `VeJoeStaking`, RocketJoe). LB versions: [`v2.0.md`](v2.0.md), [`v2.1.md`](v2.1.md), [`v2.2.md`](v2.2.md). Index: [`README.md`](README.md).
**Key fact:** V1 pair/factory **topic0s are IDENTICAL to Uniswap V2** (`uniswap/v2.md`) — only factory/router *addresses* and the **pair init code hash** differ. The **router renames every `ETH` function to `AVAX`**, so those selectors differ from Uniswap (token↔token swaps keep the Uni selectors). V1 AMM is on **Avalanche (home), Arbitrum, BNB Chain — NOT Ethereum/Base/Optimism/Polygon**.

## Cross-chain summary
- **JOE token:** Avalanche native `0x6e84…0fDd` (own bytecode, 7,087 B); **bridged JOE `0x371c…2f07` is the SAME address on Arbitrum, BSC, and Ethereum** (18,324 B). xJOE/veJOE/rJOE are Avalanche-only.
- **V1 AMM** on Avalanche, Arbitrum, BNB Chain only — three distinct factory/router addresses, identical bytecode.
- **Staking/farms** (MasterChefJoe V2/V3, BoostedMCJ, sJOE, veJOE, rJOE/RocketJoe) are **Avalanche-only**.

## Proxies
- **V1 AMM: NO proxies.** `JoeFactory`, `JoeRouter02` immutable; **`JoePair` is a full CREATE2-deployed contract** (Uniswap-V2 style — *not* a clone/minimal-proxy), salt `keccak256(token0,token1)`, Joe-specific init code hash.
- **Tokens immutable:** JOE, xJOE, veJOE, rJOE, plus `MasterChefJoeV2`/`V3` (plain contracts, no EIP-1967 impl slot).
- **Upgradeable (EIP-1967 transparent/UUPS proxies — watch for `Upgraded`):** `StableJoeStaking` (sJOE), `VeJoeStaking`, `BoostedMasterChefJoe`, `RocketJoeStaking`. Verified live via impl slot `0x360894…bbc`.

## Detection invariants & gotchas
1. **V1 pair topic0s == Uniswap V2 == Sushi V2.** Disambiguate by the **JoeFactory/pair address**, never by topic0. The init code hash is Joe-specific (≠ Uni's `0x96e8ac42…`, ≠ Sushi's `0xe18a34eb…`).
2. **Router AVAX selectors differ from Uniswap.** A monitor keyed on Uniswap's `swapExactETHForTokens` (`0x7ff36ab5`) misses Joe's `swapExactAVAXForTokens` (`0xa2a1623d`). Token↔token selectors are shared.
3. **`Deposit(address,uint256,uint256)` `0x90890809…` is emitted by BOTH MasterChefJoe AND sJOE** (and Sushi/Pancake) — the second/third fields differ in meaning (`pid,amount` vs `amount,fee`). Disambiguate by contract.
4. **`Withdraw(address,uint256,uint256)` `0xf279e6a1…` is emitted by MasterChefJoe AND veJOE staking** — sJOE's `Withdraw` is the 2-arg `0x884edad9…` instead. Mind the arity.
5. Staking contracts are **upgradeable** — an `Upgraded` event on sJOE/veJOE/BoostedMCJ/RocketJoeStaking changes their logic.

# Arrakis V2 — Topics, Selectors, Addresses (Ethereum, Polygon, Arbitrum, Optimism, Base, BNB; absent on Avalanche)

**Status:** verified against live RPC on all six deployed chains and the canonical `ArrakisFinance/v2-core`, `v2-periphery`, and `v2-manager-templates` sources on 2026-06-04. Factory confirmed by `numVaults()` view on every chain; vault event topic0s, factory `VaultCreated`, and the BeaconProxy slot cross-checked against a live mainnet vault deployment. Absence on Avalanche C-Chain verified by `eth_getCode = 0x`.

**Scope:** the **Arrakis V2** generation — the `v2-core` contracts. An **ArrakisV2 vault** is a multi-position Uniswap-V3 liquidity manager: one ERC-20 LP token whose backing is spread across **many** Uniswap-V3 ranges (`(lowerTick, upperTick, feeTier)`) simultaneously, rebalanced by an external **manager**. This is structurally different from **Arrakis V1** (the `g-uni` / GUniPool one-range-per-pool ERC-20 wrapper) and from the newer **Arrakis Modular** (V2.1, `arrakis-modular`, the `ArrakisMetaVault*` + `*Module` generation — NOT covered here). Covered contracts: `ArrakisV2Factory`, the `ArrakisV2` vault implementation + `ArrakisV2Beacon`, `ArrakisV2Resolver`, `ArrakisV2Helper`, the periphery `ArrakisV2Router` (a.k.a. RouterV2) + `RouterSwapExecutor` + `RouterSwapResolver`, and the `SimpleManager` manager template.

Orientation:

- **Vaults are factory-deployed beacon proxies.** `ArrakisV2Factory.deployVault(InitializePayload, bool isBeacon_)` deploys a new vault. With `isBeacon_ = true` the vault is an **OpenZeppelin BeaconProxy** pointing at `ArrakisV2Beacon` (so all beacon vaults share the upgradeable `ArrakisV2` implementation); with `isBeacon_ = false` it is a `TransparentUpgradeableProxy` with the same implementation but an independent admin. Every deploy emits `VaultCreated(manager indexed, vault indexed)`.
- **Enumerate all vaults via the factory.** `numVaults()` → count, `vaults(startIndex, endIndex)` → `address[]`. There is no per-vault registry event other than `VaultCreated`; subscribe to that topic on the factory to discover vaults.
- **Manager-driven rebalancing.** Each vault has a `manager` (set at init / via `setManager`). The manager — typically a `SimpleManager`, a PALM manager, or in practice often a **Gnosis Safe multisig** — calls `vault.rebalance(Rebalance)` to burn/mint Uniswap-V3 positions and (optionally) swap. The manager also collects the manager fee. Only the manager can rebalance; only whitelisted routers can be passed in the rebalance swap payload.
- **Addresses are deterministic (same on every chain).** The factory, beacon, vault implementation, resolver, helper, router, swap executor/resolver, and SimpleManager all share one address across all six chains (deployed via a deterministic deployer). Only the factory's logic-implementation differs on Gnosis (out of the seven target chains, Gnosis is not in scope).

## Detection invariants & gotchas

- **V2 ≠ V1 ≠ Modular.** V1 (`g-uni`/GUniPool) is a single-range ERC-20 wrapper with a different factory and `GelatoUniV3Pool`-style events. V2 vaults hold **many** ranges and emit the `Log*` events above. Arrakis **Modular** (V2.1) uses `ArrakisMetaVault` + pluggable `*Module` contracts and entirely different events/addresses — do not conflate.
- **`vault.factory()` is the Uniswap-V3 factory, not the ArrakisV2Factory.** To find the deploying Arrakis factory, use the fixed address `0xECb8Ffcb…` (same on all six chains).
- **Enumerate vaults from the factory**, via `numVaults()` + `vaults(start,end)` or the `VaultCreated` topic. There is no public per-vault registry beyond that.
- **Vaults are BeaconProxies by default** (impl slot empty; read the **beacon slot** `0xa3f0ad74…133d50`, then `beacon.implementation()`). Non-beacon vaults are transparent proxies (impl slot populated). A vault deployed with `isBeacon_=false` and later frozen via `makeVaultsImmutable` is non-upgradeable.
- **Only the manager can rebalance.** `vault.rebalance` (`0x8712e497`) reverts for non-managers. The manager is whatever `vault.manager()` returns — often a Gnosis Safe, sometimes `SimpleManager` (`0x3522Df5C…`) or a PALM manager. Don't assume SimpleManager.
- **Only whitelisted routers** can appear in the rebalance swap payload (`LogWhitelistRouters` / `getRouters()`); a rebalance swap through a non-whitelisted router reverts.
- **`LogMint` topic0 `0x5f118302…` is the deposit signal; `LogBurn` `0x86dacd5c…` is the withdrawal signal.** `LogRebalance` `0xab3abe7e…` is the position-management signal (manager activity). `LogCollectedFees` `0xdbbb3a79…` fires when Uniswap-V3 fees are pulled into the vault.
- **Manager fee** is in BPS: changed via `setManagerFeeBPS` on the vault (manager-only, `0x4794c84a`) emitting `LogSetManagerFeeBPS`, and withdrawn via `withdrawManagerBalance` (`0x7ecd6717`) emitting `LogWithdrawManagerBalance`.
- **SimpleManager `setManagerFee` (`0x725880e9`) emits the event named `SetManagerFeeBPS`** — the function and event names differ; match on the event topic, not the function name.
- **Deterministic addresses ≠ deployed everywhere.** The same address is used on all chains where Arrakis V2 exists, but Avalanche has **no** deployment (`0x`). Always `eth_getCode`-check before assuming presence.

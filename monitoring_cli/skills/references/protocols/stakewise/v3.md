# StakeWise V3 — Topics, Selectors, Addresses (Ethereum mainnet)

**Status:** verified against live RPC on `https://ethereum-rpc.publicnode.com` and the canonical `stakewise/v3-core` repo (`deployments/mainnet.json`) + `@stakewise/v3-sdk` `4.2.5` on 2026-06-09. Every address existence-checked with `eth_getCode` (code present); proxy/immutability confirmed by EIP-1967 slot reads; topic0s/selectors recomputed locally as keccak256 of the canonical signature (structs expanded to canonical tuples) and the high-volume ones confirmed against live `eth_getLogs` / present in deployed bytecode.
**Scope:** StakeWise **V3** (the vault / osETH model). **Of the 7 target chains it is deployed on Ethereum mainnet (chain ID 1) ONLY** — `eth_getCode` returns `0x` for every V3 address on Base, BNB, Avalanche, Arbitrum, Optimism, and Polygon. V3 also runs on **Gnosis Chain (100)** (osGNO) and the **Hoodi testnet (560048)** — both outside the 7 (Gnosis addresses listed for context). **There is no StakeWise V3 on Arbitrum**: the `v3-core` repo ships no Arbitrum deployment and the canonical addresses have no code there. Topic0/selector values are chain-agnostic; addresses are Ethereum-specific.

**What V3 is.** Anyone deploys a **Vault** via the `EthVaultFactory` (or a priv / erc20 / blocklist / meta variant). A vault holds staked ETH; depositors receive vault shares (`Deposited`), and may **mint osETH** (`OsToken`) against their position up to an LTV (`OsTokenMinted`). **osETH is a single, fungible, non-rebasing, overcollateralized token** shared across *all* vaults — its value vs ETH accrues via the `OsTokenVaultController` (`convertToAssets`). A single **Keeper** harvests every vault from one oracle-signed Merkle root (`RewardsUpdated` → per-vault `Harvested`). Withdrawals are an **exit queue** (`ExitQueueEntered` → `ExitedAssetsClaimed`). The **GenesisVault** is the V2-Pool successor (V2 sETH2/rETH2 migrate into it).

> **The key indexing fact.** Vaults are **not enumerated** — each is a per-vault **ERC-1967 proxy** (`new ERC1967Proxy(implementation, "")`) minted by a factory and registered in the `VaultsRegistry`. Discover them via **`VaultCreated`** (`0x0d606510…`, factory) and **`VaultAdded`** (`0x2f069741…`, registry). All per-vault staking/osETH/exit events are emitted **by the vault address**, so key everything on `(chainId, vaultAddress)`. The vault implementation is shared per factory version (factory `implementation()` is immutable); each vault can self-upgrade via `upgradeToAndCall` to a registry-allowlisted impl.

## 1. Topics (chain-agnostic — `topic0 = keccak256(event signature)`)

**✓** = observed verbatim in live mainnet `eth_getLogs` (49k-block window or full-history scan); **abi** = computed from canonical source, not log-confirmed this pass. `topic0` is unaffected by `indexed`.

## 5. Proxies

| Contract | Pattern | Detection | Upgrade authority |
|----------|---------|-----------|-------------------|
| **Core singletons** — VaultsRegistry, all factories, Keeper, OsToken, OsTokenVaultController, OsTokenConfig, OsTokenVaultEscrow, OsTokenFlashLoans, OsTokenRedeemer, DepositDataRegistry, PriceFeed, SharedMevEscrow, RewardSplitterFactory, MerkleDistributor, CuratorsRegistry | **Immutable (no proxy)** | EIP-1967 impl slot `0x3608…2bbc` reads `0x0`; redeployed (not upgraded) — a new factory ships a new vault impl | redeploy + re-register in VaultsRegistry |
| **Vaults** — GenesisVault, FoxVault, CommunityVault, every factory-minted vault | **ERC-1967 proxy** | impl slot non-zero (e.g. GenesisVault → `0xf113bfd6…`); proxy bytecode is the minimal OZ `ERC1967Proxy` (~260 B) | vault admin via `upgradeToAndCall` to a `VaultsRegistry`-allowlisted impl |

**Detection.** The split is the **opposite of V2**: in V3 the *infrastructure* is immutable and the *vaults* are upgradeable. To find a vault's current logic, read the standard EIP-1967 impl slot on the **vault**, not on the registry/keeper/osToken (those are `0x0`). Vault upgrades emit the OZ `Upgraded(address indexed implementation)` topic `0xbc7cd75a20ee27fd9adebab32041f755214dbc6bffa90cc0225b39da2e5c2d3b`; an allowlisting precedes them (`VaultImplAdded` on the registry).

## 6. Detection invariants & gotchas

1. **Vaults are discovered, never hardcoded.** Index `VaultCreated` (factory `0x0d606510…`) + `VaultAdded` (registry `0x2f069741…`) to enumerate vaults; **all per-vault events are emitted by the vault address.** Key on `(chainId, vaultAddress)`. The default factory plus four variant factories (priv/erc20/blocklist/meta) each emit the *same* `VaultCreated` topic0 from *different* addresses.
2. **osETH is overcollateralized debt, not a share token.** `OsTokenMinted` is a user borrowing osETH against a vault position; the global osETH supply changes via `OsTokenVaultController.Mint`/`Burn`. To value osETH use `OsTokenVaultController.convertToAssets` / `PriceFeed`, **not** a 1:1 assumption. osETH does **not** rebase.
3. **`Harvested` is three different events; `MevReceived` is shared.** Keeper `Harvested(address,bytes32,int256,uint256)` `0x3f1d24da…` vs SharedMevEscrow `Harvested(address,uint256)` `0x121c5042…` vs OwnMevEscrow `Harvested(uint256)` `0x8e55ccfc…`. `MevReceived(uint256)` `0x7cb3607a…` fires from SharedMevEscrow **and** every per-vault OwnMevEscrow. Disambiguate by emitter.
4. **`OsTokenLiquidated`/`OsTokenRedeemed`/`ExitedAssetsClaimed`/`ExitQueueEntered` recur with different tuples** across the vault, the OsTokenRedeemer (`0xc43A7b16…`), and the OsTokenVaultEscrow (`0x09e84205…`). The vault tuples (§1.1/1.2) and the escrow tuples (§1.8) differ — decode against the **emitter's** ABI.
5. **The daily harvest is `Keeper.RewardsUpdated`** (`0xfa453467…`, one per oracle report) → per-vault `Harvested` with `totalAssetsDelta` (negative = slashing). This is the protocol-health pulse, analogous to a rebase.
6. **GenesisVault is the V2 bridge.** It is a normal V3 vault but predeployed (not via factory, so no `VaultCreated`); it emits `Migrated` for incoming V2 positions and references the shared `LegacyPoolEscrow` `0x2296e1…`. Don't miss it by filtering only factory-created vaults.
7. **`OracleAdded`/`OracleRemoved` collide with V2 Oracles.** Same topic0s — Keeper `0x6B581546…` vs V2 Oracles `0xF0C1670364…`. See [v2.md](v2.md).
8. **Core singletons are immutable; only vaults upgrade.** Reading the EIP-1967 impl slot on the Keeper/registry/osToken returns `0x0` — that's expected, not an error. Watch `Upgraded` only on vault addresses (and on the V2 token proxies in [v2.md](v2.md)).
9. **Single-chain among the 7.** No bridged osETH on any L2 target; an "osETH" on Base/Arb/etc. is unrelated. The Gnosis osGNO is a separate xDAI deployment. The factory address `0x7A8c…22DD` exists on both Ethereum and Gnosis — key on `(chainId, address)`.
10. **ERC-20 vaults add token events.** `Erc20Vault` instances also emit ERC-20 `Transfer`/`Approval` for their share token (alongside `Deposited`) — the same `0xddf252ad…` topic0 as osETH and every other ERC-20; disambiguate by address.

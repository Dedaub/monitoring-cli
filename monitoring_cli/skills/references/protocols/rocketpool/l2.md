# Rocket Pool on L2s & Sidechains — Bridged rETH (Arbitrum, Optimism, Base, Polygon, BNB, Avalanche)

**Status:** verified 2026-05-29 via `eth_getCode` / `name()` / `symbol()` / `decimals()` / EIP-1967 slot reads on each chain's `*-rpc.publicnode.com`, cross-checked with docs.rocketpool.net and the `rocket-pool/rocketpool-*-oracle` repos. Token/event signatures are chain-agnostic; addresses are network-specific.

**Scope:** Rocket Pool's **non-Ethereum footprint = the bridged rETH token only**. The core staking protocol (minipools, megapools, RPL staking, RocketStorage, oracles) lives **only on Ethereum** — see [core.md](core.md). There is **no staking, no minting, no node operation, no core contract** on any chain below; rETH is minted/burned exclusively on L1 and moved cross-chain by bridges.

> **Key fact:** Rocket Pool **does not run its own bridges**. Every L2 rETH was pushed through the chain's **own canonical bridge** by the community/third parties; per the *Rocket Pool Bridge Campaign* governance thread, *"there is a single canonical rETH, which is the native bridge"* and *"the DAO does not own the CA on Arbitrum."* What the DAO **does** run off-mainnet are **rate oracles** (rETH/ETH exchange rate, pushed from L1) — explicitly *"provided as-is… not formally audited."*
>
> **rETH NEVER rebases on any chain** (value-accruing like wstETH). Bridged rETH carries **no on-chain rate** — read the per-chain rate oracle (§3) or convert off the L1 `rocketTokenRETH.getExchangeRate()`.

## 1. Topics (chain-agnostic)

Bridged rETH is a standard ERC-20 (+ bridge mint/burn). Token mint/burn surfaces as `Transfer` from/to `0x0`. **There are no Rocket Pool staking events on these chains** (no `TokensMinted`/`TokensBurned`/`EtherDeposited` — those are L1-only on `rocketTokenRETH`; see [core.md](core.md) §2.1).

| topic0 | Event | ✓ |
|--------|-------|---|
| `0xddf252ad1be2c89b69c2b068fc378daa952ba7f163c4a11628f55a4df523b3ef` | `Transfer(address,address,uint256)` | abi |
| `0x8c5be1e5ebec7d5bd14f71427d1e84f3dd0314c0f7b2291e5b200ac8c7c3b925` | `Approval(address,address,uint256)` | abi |

Bridge-side events vary by chain (OP-Stack `DepositFinalized`/`WithdrawalInitiated` on the StandardBridge predeploy `0x4200…0010`; Arbitrum gateway events; Polygon `ChildChainManager` deposits). Rate-oracle events (e.g. `RateUpdated`) vary by oracle version — pull the live ABI before alerting.

## 5. Detection invariants & gotchas

1. **No staking/minting/oracle-of-record off Ethereum.** All stake/unstake/validator events are L1-only ([core.md](core.md)). These chains only `Transfer` bridged rETH.
2. **Bridged rETH never rebases and carries no rate** — for ETH value use the chain's `Rocket<Chain>PriceOracle.rate()` or the L1 `getExchangeRate()`. Don't assume 1 rETH = 1 ETH.
3. **rETH addresses differ per chain** — no address is reused across chains (unlike some LayerZero OFTs). Match by chain ID.
4. **Polygon token is "(PoS)"-suffixed and is a Polygon-bridge child token**, not a Rocket Pool contract; its proxy is Polygon's, not EIP-1967.
5. **The DAO owns none of these token CAs.** Report L2 rETH as canonical-bridge/third-party, not "official Rocket Pool deployment." Rate oracles are the only DAO-run, explicitly as-is/unaudited, contracts.
6. **BNB & Avalanche: no canonical rETH.** Flag the Avalanche `0x7E6Bc9…24de0` address as bogus if encountered.

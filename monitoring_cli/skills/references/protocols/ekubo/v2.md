# Ekubo Protocol V2 (EVM) — Topics, Selectors, Addresses (Ethereum + Sepolia, DEPRECATED)

**Status:** **DEPRECATED** — superseded by [`v3.md`](./v3.md). Verified against live Ethereum RPC and the `EkuboProtocol/evm-contracts` **`v2.1.0`** tag on 2026-06-05. All addresses confirmed live via `eth_getCode`; the two high-frequency Core topics confirmed against live logs; all hashes recomputed locally.
**Scope:** Ekubo's **first public EVM deployment (V2)** — **Ethereum mainnet + Sepolia only** ("the contracts are not deployed to any other EVM chain"). There is **no V1** EVM mainnet (the numbering tracks the Starknet protocol generation). V2 is the **same architecture family** as V3 (singleton `Core`, extensions, `lock()` flash accounting, anonymous swap logs) but predates the **`MEVResist`→`MEVCapture` rename**, uses an explicit **`Bounds`/`PositionKey`** position model (V3 packs a `PositionId` bytes32), and keeps positions/protocol-fees logic inside `Core` (V3 splits more into periphery).

Everything immutable; **no proxies, no upgrade mechanism** — V2→V3 was a full redeploy, not an upgrade.

> **Use V3 for anything live.** This file exists because the V2 contracts are still on-chain and may still emit events / hold dust. For the maintained multichain surface, see [`v3.md`](./v3.md).

## 1. Topics (chain-agnostic — `topic0 = keccak256(event signature)`)

### 1.1 Core (`0xe0e0…d444`) — named events

`PositionUpdated` (76×, 256-byte data) and `PositionFeesCollected` (39×, 224-byte data) confirmed against live Ethereum `Core` logs (56k-block window); the other three are recomputed and **share the V3 hash** (identical signatures).

| topic0 | Event | vs V3 |
|--------|-------|-------|
| `0x5e4688b340694b7c7fd30047fd082117dc46e32acfbf81a44bb1fac0ae65154d` | `PoolInitialized(bytes32 poolId, PoolKey poolKey, int32 tick, SqrtRatio sqrtRatio)` | **same** |
| `0xf7e050d866774820d81a86ca676f3afe7bc72603ee893f82e99c08fbde39af6c` | `FeesAccumulated(bytes32 poolId, uint128 amount0, uint128 amount1)` | **same** |
| `0xec1256266e470abb868620c851f6bde2a3ff602549dcad318ab9ccfcb2977f14` | `ExtensionRegistered(address extension)` | **same** |
| `0xa2d4008be4187c63684f323788e131e1370dbc2205499befe2834005a00c792c` | `PositionUpdated(address locker, bytes32 poolId, UpdatePositionParameters params, int128 delta0, int128 delta1)` | **V2-only** |
| `0xbb3992d83c721f12a8f32242e0d21c3613949c6a69d2a35deecdf6943a61c8b2` | `PositionFeesCollected(bytes32 poolId, PositionKey positionKey, uint128 amount0, uint128 amount1)` | **V2-only** |
| `0x8fc241308ffc17817e6a8c6a52a8f7cd4931dfca0c539fd35a630311c7e4c57b` | `ProtocolFeesWithdrawn(address recipient, address token, uint256 amount)` | **V2-only** |

Where `UpdatePositionParameters = (bytes32 salt, Bounds bounds, int128 liquidityDelta)`, `Bounds = (int32 lower, int32 upper)`, `PositionKey = (bytes32 salt, address owner, Bounds bounds)`, `PoolKey = (address token0, address token1, Config config)` and `Config` is `bytes32` (packs `extension`‖`fee`‖`tickSpacing`, like V3's `PoolConfig`). **V2's position events carry the `(salt, bounds[, owner])` tuple in full; V3 replaced it with a single packed `PositionId` bytes32** — this is the core V2↔V3 incompatibility. No fields are indexed (all in `data`).

### 1.2 Anonymous swap log (NO `topic0`) — 116 bytes

Confirmed live: 2240/2240 zero-topic `Core` logs were exactly 116 bytes. Same gas trick as V3, but the trailing state word is laid out as **explicit fields** (V3 packs them into one `PoolState`). `Core.sol:580-586`.

| offset | field |
|--------|-------|
| `[0:20]` | locker address |
| `[20:52]` | `PoolId` (bytes32) |
| `[52:68]` | `int128 delta0` |
| `[68:84]` | `int128 delta1` |
| `[84:100]` | `uint128 liquidity` (after swap) |
| `[100:112]` | `uint96 sqrtRatio` (after swap) |
| `[112:116]` | `int32 tick` (after swap) |

Detect a V2 swap: `address == 0xe0e0…d444 AND topics == [] AND length(data) == 116`.

## 4. Proxies

**None.** All V2 contracts are immutable; `eth_getStorageAt(Core, 0x360894…bbc)` = `0x0`. There is no upgrade mechanism — the "upgrade" to V3 is a separate deployment at new addresses (§3 here vs `v3.md §4`).

## 5. Detection invariants & gotchas (V2-specific)

1. **Swaps are anonymous `log0` (no topic), 116 bytes** — same detection rule as V3 but the trailing state layout differs (§1.2). Filter `address == 0xe0e0…d444 AND topics == []`.
2. **`swap_611415377` selector = `0x00000000`** (mined), but unlike V3 it takes **explicit ABI args**, and the V2 lock callback `locked(uint256)` is `0xb45a3c0e` (NOT zero — V3 mined that to `0x00000000`).
3. **V2 position events ≠ V3.** `PositionUpdated` (`0xa2d4…792c`) and `PositionFeesCollected` (`0xbb39…c8b2`) carry full `(salt, bounds[, owner])` tuples; V3 uses packed `PositionId`. Do **not** apply V3 decoders to V2 logs. `PoolInitialized`/`FeesAccumulated`/`ExtensionRegistered`/`OrderUpdated`/`OrderProceedsWithdrawn` **are** identical across V2/V3.
4. **`ProtocolFeesWithdrawn` (`0x8fc2…c57b`) is V2-only** (V2 keeps protocol fees in `Core`; V3 moved them to `Positions`).
5. **Ethereum + Sepolia only.** Any "Ekubo on Base/Arb/…" is V3, never V2. Native token = `address(0)` (same as V3).

## 7. V2 → V3 migration cheatsheet

| Aspect | V2 | V3 |
|--------|----|----|
| Chains | Ethereum + Sepolia only | ETH, Base, BNB, Arb, OP, Polygon, MegaETH, Monad, Ink, Unichain (not Avalanche) |
| MEV extension | `MEVResist` / `MEVResistRouter` | `MEVCapture` / `MEVCaptureRouter` |
| Position identity | `(salt, Bounds(int32,int32)[, owner])` tuples in events/calls | packed `PositionId` bytes32 |
| `PositionUpdated` topic | `0xa2d4…792c` | `0x704b…0a07` |
| `PositionFeesCollected` topic | `0xbb39…c8b2` | `0xd76e…7753` |
| swap fn | `swap_611415377(args)` (zero-sel, explicit args) | `swap_6269342730()` (zero-sel, raw calldata) |
| lock callback | `locked(uint256)` = `0xb45a3c0e` | `locked_6416899205(uint256)` = `0x00000000` |
| flash helpers | `load` / `save` | `startPayments` / `completePayments` / `updateSavedBalances` |
| protocol fees | in `Core` (`ProtocolFeesWithdrawn`) | in `Positions` (`withdrawProtocolFees`) |
| extras | — | `Incentives`, `BoostedFees`, `ManualPoolBooster`, `Auctions`, `RevenueBuybacks`, many data fetchers |
| **Same across both** | `PoolInitialized`, `FeesAccumulated`, `ExtensionRegistered`, `OrderUpdated`, `OrderProceedsWithdrawn`; `lock()`, `initializePool`, `accumulateAsFees`, `registerExtension`, `sload`/`tload`; anonymous 116-byte swap log; native = `address(0)`; immutable/no-proxy | |

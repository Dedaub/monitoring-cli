# Seamless Protocol V1 (legacy Aave-V3-fork lending market) — Topics, Selectors, Addresses (Base only)

**Status:** verified against Base mainnet RPC (`https://base-rpc.publicnode.com`) and the canonical `seamless-protocol/*` GitHub repos (`lending-deployment`, `aave-v3-core`, `seamless-interface`) on 2026-06-08. Non-obvious claims additionally cross-checked against primary sources — see §10.
**Scope:** the original Seamless lending market — a **soft fork of Aave V3 (V3.0-era)** deployed **only on Base (chain ID 8453)**. `eth_getCode` = `0x` for every Seamless V1 address on Ethereum, BNB, Avalanche, Arbitrum, Optimism and Polygon — this market has **never** been deployed on any chain other than Base. Topics + selectors are chain-agnostic (and identical to Aave V3 — see [aave/v3.md](../aave/v3.md)); addresses are Base-specific. The newer Morpho-powered products (Leverage Tokens, Seamless Morpho Vaults) are a separate product line — see [leveragetokens.md](leveragetokens.md) and [README.md](README.md).

Seamless is a **fully proxied, upgradeable** Aave-V3 fork (the longest-running native DeFi protocol on Base, live since 2023). Every core contract (Pool, PoolConfigurator, aTokens, debt tokens) sits behind Aave's `InitializableImmutableAdminUpgradeabilityProxy` (EIP-1967 impl slot, immutable admin). The `Pool` emits all supply/borrow/repay/liquidation events; per-reserve **aTokens are branded with an `s` prefix** (sUSDC, sWETH, swstETH — `s` = "Seamless", `name()` = e.g. "Seamless USDC") but are mechanically identical Aave aTokens. The market id is **"Base Seamless Market"** (read live from `PoolAddressesProvider.getMarketId()`).

> **DEPRECATED / WINDING DOWN (verified 2026-06-08).** On **2026-04-07** the Seamless team announced an orderly wind-down (cited insufficient product-market fit for Leverage Tokens, liquidity constraints, no sustainable revenue). The web app goes **offline 2026-06-30**; users were told to withdraw before then ("manual contract interaction" required afterward). The V1 lending contracts remain **live on-chain and functional** (read 2026-06-08) but are in withdraw-only twilight — earlier Seamless migrated core lending activity to Morpho ("Seamless 2.0"), so V1 is the *legacy* market. Treat new V1 activity as residual / migration-related, not organic growth. The Morpho Vaults are also being wound down (curator **Gauntlet** pulling funds → APY 0%) and **stkSEAM** staking is being switched off.

> **Version note (verified on-chain 2026-06-08):** the live Pool impl `0x16e48501634Dfbb52F556430d788c110E5bDA6Cf` (= the `lending-deployment` broadcast `Pool` deploy) is an **Aave V3.0-era** Pool (21,754 bytes). Bytecode selector scan: `supply`/`borrow`/`repay`/`withdraw`/`flashLoan`/`flashLoanSimple`/`supplyWithPermit`/`mintToTreasury` are **present**; the Aave **v3.1** Position-Manager, **v3.3** deficit (`getReserveDeficit`/`DeficitCreated`), and **v3.4** `multicall` additions are **all absent**. Stable-rate code is present-but-disabled by config. There is no Seamless "V2" lending Pool — the successor is Morpho-based, not a forked Pool redeploy.

> **Liquidation dispatcher anomaly (verified, same as Aave/Spark):** the canonical `liquidationCall` selector `0x00a718a9` is **absent from the live Pool impl bytecode** (raw scan = 0 occurrences) even though `supply`/`borrow`/`flashLoan` are present, yet `LiquidationCall` events fire normally (555 logs in a 49k-block window at block ~18.0M). **Detect Seamless V1 liquidations by the `LiquidationCall` event topic0 `0xe413a321…`, never by the function selector.** See §8.2.

## 7. Proxies

### 7.2 The `liquidationCall` dispatcher gotcha (verified, same as Aave/Spark)

On the live Pool impl `0x16e48501…` (21,754 bytes), selector `0x00a718a9` (`liquidationCall`) is **absent from the bytecode** (raw scan returns 0), while `supply` (`617ba037`), `borrow` (`a415bcad`), `flashLoan` (`ab9c4b5d`) are present. ## 8. Detection invariants & gotchas

1. **Seamless V1 lending lives on Base only** (Pool `0x8F44…75a7`). It returns `0x` on all six other requested chains, and it is **deprecated / winding down** (app offline 2026-06-30, withdraw-only). New activity is residual.
2. **It's an Aave V3.0 fork** — `Supply`/`Borrow`/`Repay`/`Withdraw`/`LiquidationCall`/`FlashLoan` topics and selectors are identical to [aave/v3.md](../aave/v3.md). Reuse that detection set.
3. **No v3.3/v3.4 features.** Don't scan for `DeficitCreated`/`DeficitCovered`, `multicall`, `approvePositionManager`, `getReserveDeficit`, `getReserveAToken` — absent (§2.6). No Portal `MintUnbacked`/`BackUnbacked` either.
4. **Detect liquidations by the `LiquidationCall` event, not the selector** (§7.2) — selector absent from impl; bots route liquidations so `tx.to` ≠ Pool.
5. **aTokens are "sTokens" (sUSDC, sWETH, …) — `s` = Seamless.** They rebase every block. Store `scaledBalanceOf` + reconstruct with the reserve `liquidityIndex` (ray). `Mint`/`Burn` topic0 is shared between sToken and debt token — disambiguate by emitting address.
6. **`onBehalfOf` ≠ `tx.from`.** Attribute positions to `onBehalfOf`/`user` from the event, not the sender (WETHGateway, credit delegation, relayers).
7. **`CapsPlusRiskSteward` (`0x789aAeF4…`) fires `SupplyCapChanged`/`BorrowCapChanged`** automatically (not only governance). During the wind-down, expect caps being driven toward zero.
8. **Governance = Timelock Short `0x639d2dD2…`** (ACL_ADMIN / POOL_ADMIN, the Seamless DAO short timelock). Two-tier governance (Timelock/Governor Short + Long) — see [README.md](README.md).
9. **SEAM is both a reserve and the gov token.** `0x1C7a4604…` is supplyable/borrowable in this market *and* the governance token. Don't conflate market activity with token transfers.
10. **Reserve caps/configs are being zeroed during wind-down.** Always read the current reserve set live (`getReservesList` + `getReserveData`); the listed set may shrink before 2026-06-30.

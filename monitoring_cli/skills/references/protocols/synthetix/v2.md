# Synthetix V2 — Topics, Selectors, Addresses (Ethereum, Optimism)

**Status:** Legacy / maintenance mode. V2 is no longer being actively developed; Synthetix V3 is the active codebase. V2 contracts remain live and fully funded on both Ethereum and Optimism. ETH staking (SNX → sUSD debt) is still operational but low-activity. PerpsV2 on Optimism was the most actively traded surface until V3 migration began.

**Scope:** Ethereum mainnet (chain 1) — SNX token, sUSD staking, synth exchange, SNX bridge; Optimism (chain 10) — bridged SNX, sUSD, PerpsV2 markets, same staking stack. No V2 deployments on BNB Smart Chain (56), Avalanche (43114), Arbitrum One (42161), Base (8453), or Polygon PoS (137) — verified via `eth_getCode` returning `0x`.

---

## Architecture

**AddressResolver** (`AddressResolver.sol`) is the central registry: every V2 contract holds a reference to the resolver and looks up its peers by `bytes32` name key at startup or via `resolver.getAddress(name)`. Upgrading a component means deploying a new implementation, registering it in the resolver, and calling `rebuildCaches(address[])` on all dependents.

**Proxyable pattern.** Most user-facing V2 contracts are wrapped by a thin `Proxy` contract (Synthetix's custom proxy, not EIP-1967). The proxy stores the implementation address — referred to as the `target` — at **storage slot 2** (owner = slot 0, nominatedOwner = slot 1, target = slot 2). The public `target()` view function reads slot 2. Events are emitted with `msg.sender = proxy`, so all event filters must use the **proxy address**, not the implementation address. There is no EIP-1967 slot; `eth_getStorageAt(proxy, 0x360894...)` returns zero.

**ProxyPerpsV2 (different pattern).** PerpsV2 market proxies (`0x59b0...`, `0x2B3b...`, etc.) use a router-based pattern, not the simple Proxyable proxy. Each proxy stores a `RouteData[]` array at storage slot 2 (length) with array elements at `keccak256(abi.encode(2))`. Each `RouteData` struct (packed in one slot) holds a `bytes4` selector, a `address` implementation, and a `bool isView`. The proxy dispatches each call to the implementation that registered that selector. There are currently five implementation contracts per market.

**PerpsV2 V2 is in maintenance mode.** The markets are no longer accruing new open interest after the V3 migration of collateral. All markets still exist and can be monitored; `PositionModified` and related events stop being emitted when positions close or expire but the contracts are unpaused.

## 7. Proxies

### 7.1 Proxyable (SNX token, sUSD, other synths)

Custom Synthetix proxy. **Not** EIP-1967.

```
Storage layout:
  slot 0  → owner (address)
  slot 1  → nominatedOwner (address)
  slot 2  → target / implementation (address)
```

- Read implementation: `cast call <proxy> "target()(address)" --rpc-url <rpc>`
- Or directly: `eth_getStorageAt(<proxy>, "0x2")`
- Events are emitted from the **proxy address** because the implementation calls `emitX(...)` which emits via the proxy using the `Proxyable` base's `messageSender` relay.
- No EIP-1967 slot (`0x360894...bbc` returns zero).

Affected contracts: ProxySynthetix (ETH + OPT), ProxysUSD (ETH + OPT), all synth proxies (e.g. sEUR, sBTC synths when they existed).

### 7.2 ProxyPerpsV2 (PerpsV2 markets)

Router/diamond-like pattern. **Different from Proxyable.**

```
Storage layout:
  slot 0  → owner (address)
  slot 1  → nominatedOwner (address)
  slot 2  → routes.length (uint256, e.g. 0x30 = 48)
  slot 3  → (unused)
  slot 4  → (unused)
  slot 5  → numImplementations (uint256, e.g. 5)

Array base: keccak256(abi.encode(uint256(2)))
  = 0x405787fa12a823e0f2b7631cc41b3ba8828b3321ca811111fa75cd3aa3bb5ace

Each element (RouteData, packed in 1 slot):
  bits  [0:31]    → selector (bytes4, right-aligned)
  bits [32:191]   → implementation (address, 20 bytes)
  bits [192:199]  → isView (bool, 1 byte)
```

- No `target()` function. To find the implementation for a given selector, read the routes array and decode.
- Events are emitted from the **proxy address** (the proxy forwards calls via `delegatecall`-style routing, but events carry the proxy as `address`).

## 8. Detection invariants & gotchas

1. **Events from proxy, not impl.** For Proxyable contracts (SNX, sUSD), filter logs by the **proxy address** (`0xC011a73ee...`, `0x57ab1ec...`, etc.). The impl address will produce zero matching logs in practice because all state-changing calls go through the proxy.

2. **ProxyPerpsV2 has no `target()`.** Cannot use a simple `target()` call to find PerpsV2 implementations. Decode the routes array as described in §7.2 or use `eth_getCode` on known impl addresses.

3. **PositionLiquidated has 8 params, not 6.** The interface changed during the PerpsV2 lifecycle. The active topic0 is `0x8e83cfbf...` (8 params, with flaggerFee/liquidatorFee/stakersFee breakdown). The 6-param version (`0x62e7eb66...`) does not appear in on-chain logs for PerpsV2.

4. **PositionFlagged (not FuturesPositionFlaggedForLiquidation).** The live event name on OPT is `PositionFlagged(uint256,address,address,uint256,uint256)` → `0x68f13e2f...`, not `FuturesPositionFlaggedForLiquidation` → `0x05598065...`. The latter is absent from on-chain logs.

5. **SynthExchange emitted from ProxySynthetix.** Even though exchange logic lives in `Exchanger`, the SynthExchange event fires from the SNX proxy address (verified OPT block range 0x68E7480–0x68F0000). Filtering by `Exchanger` address yields nothing.

6. **Issued/Burned emitted from ProxysUSD.** Both ETH (0x57ab1ec...) and OPT (0x8c6f28f2...) sUSD proxies emit these, not the Synthetix proxy.

7. **FuturesMarketManager.allMarkets() returns empty.** V3 migration cleared the FMM registry on Optimism. Market proxies are still deployed and functional but enumeration via FMM no longer works. Use the deployment JSON or hard-code known proxy addresses.

8. **V2 ETH is mostly dormant.** SNX transfers (Transfer events) still occur regularly (staking rewards, transfers). SynthExchange and Issued/Burned are rare. No PerpsV2 on ETH.

9. **AddressResolver lookup = bytes32 name, right-padded with zeros.** `getAddress("Synthetix")` uses `0x53796e74686574697800000000000000000000000000000000000000000000`. Python: `name.encode().hex().ljust(64, '0')`.

10. **Synth V2 is NOT on Synthetix V3 chains.** Arbitrum, Base, and Polygon have Synthetix V3 deployments (different contracts, different ABIs). Do not use V2 addresses on those chains.

## 10. Verification & sources

**On-chain verification method.** All addresses verified via `eth_getCode` returning non-zero bytecode at the listed address on the stated chain. Proxy `target()` values verified via direct `eth_call`. Event topic0 values verified by: (a) computing `keccak256(signature_string)` with `cast keccak`, and (b) confirming the hash appears in actual on-chain logs retrieved via `eth_getLogs` from publicnode RPC endpoints.

**RPCs used:**
- Ethereum: `https://ethereum-rpc.publicnode.com`
- Optimism: `https://optimism-rpc.publicnode.com`
- BNB/Avalanche/Arbitrum/Base/Polygon: respective publicnode endpoints (absence confirmed)

**Authoritative sources:**
- Synthetix V2 GitHub: `https://github.com/Synthetixio/synthetix` — `publish/deployed/mainnet/deployment.json` (ETH), `publish/deployed/mainnet-ovm/deployment.json` (OPT)
- Synthetix docs (V2 legacy): `https://docs.synthetix.io`
- On-chain block ranges used for log verification: ETH 0x13F0000–0x13FF000 and 0xF00000–0xF0C350; OPT 0x68E7480–0x68F0000 and 0x6800000–0x6880000
